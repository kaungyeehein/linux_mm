# 2.4 Creating, Moving and Deleting Files

## သင်ခန်းစာ အညွှန်း (Reference to LPI objectives)
* **Linux Essentials version 1.6, Exam 010, Objective 2.4**
* **အလေးချိန် (Weight):** 2

### အဓိက သိရှိရမည့် နယ်ပယ်များ (Key knowledge areas)
* Files များနှင့် Directories များကို ဖန်တီးခြင်း၊ ရွှေ့ပြောင်းခြင်းနှင့် နာမည်ပြောင်းခြင်း
* Files များနှင့် Directories များကို ဖျက်ခြင်း
* Globbing (Wildcards) များကို အသုံးပြုခြင်း

---

### နိဒါန်း (Introduction)
Linux system တစ်ခုကို ထိရောက်စွာ အသုံးပြုနိုင်ရန် ဖိုင်များနှင့် ဒိုင်ရက်ထရီများကို စီမံခန့်ခွဲတတ်ရန် လိုအပ်သည်။ ဤသင်ခန်းစာတွင် ဖိုင်အသစ်များ ဖန်တီးပုံ၊ ၎င်းတို့ကို နေရာရွှေ့ပြောင်းပုံ၊ နာမည်ပြောင်းပုံနှင့် မလိုအပ်သော ဖိုင်များကို ဘေးကင်းစွာ ဖျက်ဆီးပုံတို့ကို လေ့လာပါမည်။

---

### Creating Files with `touch` (ဖိုင်အသစ် ဖန်တီးခြင်း)
`touch` command သည် အခြေခံအားဖြင့် ဖိုင်တစ်ခု၏ "နောက်ဆုံးဝင်ရောက်ခဲ့သော အချိန်" (timestamp) ကို update လုပ်ရန် သုံးသော်လည်း၊ ဖိုင်မရှိသေးပါက ဖိုင်အသစ်တစ်ခုကို အလွယ်တကူ တည်ဆောက်ပေးနိုင်သည်။
* **ဥပမာ:** `touch newfile.txt` (ဗလာဖိုင်အသစ်တစ်ခု ဖန်တီးမည်)

---

### Creating Directories with `mkdir` (ဒိုင်ရက်ထရီ ဖန်တီးခြင်း)
Directory (Folder) အသစ်တစ်ခု တည်ဆောက်ရန် `mkdir` (Make Directory) ကို သုံးသည်။
* **ဥပမာ:** `mkdir my_folder`
* **`-p` option:** လမ်းကြောင်းတစ်ခုလုံးတွင် မရှိသေးသော directory အဆင့်ဆင့်ကို တစ်ပြိုင်တည်း တည်ဆောက်ရန် သုံးသည်။
    * **ဥပမာ:** `mkdir -p project/src/data`

---

### Moving and Renaming with `mv` (နေရာရွှေ့ခြင်းနှင့် နာမည်ပြောင်းခြင်း)
Linux တွင် နေရာရွှေ့ခြင်းနှင့် နာမည်ပြောင်းခြင်း နှစ်ခုလုံးအတွက် `mv` (Move) command တစ်ခုတည်းကိုသာ သုံးသည်။

1.  **Moving (နေရာရွှေ့ခြင်း):** ဖိုင်ကို directory တစ်ခုအတွင်းသို့ ရွှေ့ခြင်း။
    * **ဥပမာ:** `mv file.txt my_folder/`
2.  **Renaming (နာမည်ပြောင်းခြင်း):** ဖိုင်ကို တည်နေရာမပြောင်းဘဲ အမည်သစ်ပေးခြင်း။
    * **ဥပမာ:** `mv oldname.txt newname.txt`

---

### Copying Files and Directories with `cp` (ဖိုင်များ ကူးယူခြင်း)
ဖိုင်တစ်ခုကို မူရင်းမပျက်ဘဲ မိတ္တူပွားလိုပါက `cp` (Copy) ကို သုံးသည်။
* **ဥပမာ:** `cp source.txt destination.txt`
* **`-r` option (Recursive):** Directory တစ်ခုလုံးနှင့် ၎င်းအတွင်းရှိ အရာအားလုံးကို ကူးယူလိုပါက သုံးရမည်။
    * **ဥပမာ:** `cp -r folder1/ folder2/`

---

### Removing Files and Directories with `rm` (ဖျက်ဆီးခြင်း)
**သတိပြုရန်:** Linux command line မှ ဖျက်လိုက်သော ဖိုင်များသည် အမှိုက်ပုံး (Trash) ထဲသို့ မရောက်ဘဲ အပြီးတိုင် ပျက်စီးသွားတတ်သည်။
* **`rm file.txt`:** ဖိုင်တစ်ခုကို ဖျက်ခြင်း။
* **`rm -r directory_name`:** Directory တစ်ခုလုံးကို ဖျက်ခြင်း။
* **`rm -i`:** မဖျက်ခင် အတည်ပြုချက် (Confirmation) တောင်းခံခြင်း။
* **`rmdir`:** အထဲတွင် ဘာမှမရှိသော (Empty) directory များကိုသာ ဖျက်ရန် သုံးသည်။

---

### Globbing / Wildcards (အစုအဖွဲ့လိုက် ကိုင်တွယ်ခြင်း)
ဖိုင်အများအပြားကို တစ်ပြိုင်တည်း ခိုင်းစေလိုသောအခါ Wildcards များကို သုံးသည်။

* **`*` (Asterisk):** မည်သည့် စာလုံးအရေအတွက်မဆို (သုညအပါအဝင်) ကို ကိုယ်စားပြုသည်။
    * **ဥပမာ:** `rm *.log` (.log နှင့် ဆုံးသမျှ ဖိုင်အားလုံးကို ဖျက်မည်)
* **`?` (Question Mark):** စာလုံး တစ်လုံးတည်း (Single character) ကို ကိုယ်စားပြုသည်။
    * **ဥပမာ:** `ls file?.txt` (file1.txt, fileA.txt စသည်တို့ကို ပြမည်)
* **`[]` (Square Brackets):** ကွင်းထဲတွင် ပါဝင်သော စာလုံးတစ်လုံးချင်းစီကို ရှာသည်။
    * **ဥပမာ:** `ls file[1-3].txt` (file1, file2, file3 တို့ကိုသာ ပြမည်)

---

### အကျဉ်းချုပ် (Summary)
ယခု သင်ခန်းစာတွင် သင်ယူခဲ့ရသည်များမှာ-
* `touch` နှင့် `mkdir` ဖြင့် ဖိုင်နှင့် directory အသစ်များ တည်ဆောက်ပုံ။
* `mv` ကို သုံး၍ နေရာရွှေ့ခြင်းနှင့် နာမည်ပြောင်းခြင်း။
* `cp` ဖြင့် မိတ္တူပွားခြင်း (Directory များအတွက် `-r` သုံးရန်)။
* `rm` ဖြင့် အပြီးတိုင် ဖျက်ဆီးပုံနှင့် သတိထားရမည့်အချက်များ။
* Wildcards (`*`, `?`, `[]`) များဖြင့် ဖိုင်များကို အမြန်နှုန်းဖြင့် စုပေါင်းကိုင်တွယ်ပုံ။
