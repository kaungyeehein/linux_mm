### နိဒါန်း (Introduction)
Linux system တစ်ခုတွင် အလုပ်လုပ်ရာ၌ Graphical User Interface (GUI) သည် အသုံးဝင်သော်လည်း၊ Command Line Interface (CLI) သို့မဟုတ် **Shell** သည် ပိုမိုမြန်ဆန်ပြီး အစွမ်းထက်ပါသည်။ Shell ဆိုသည်မှာ အသုံးပြုသူ ရိုက်နှက်လိုက်သော command များကို လက်ခံပြီး Operating System က နားလည်အောင် ဘာသာပြန်ပေးသော program ဖြစ်သည်။ Linux တွင် အသုံးအများဆုံး Shell မှာ **Bash** (Bourne Again Shell) ဖြစ်သည်။

### Command Line သို့ ဝင်ရောက်ခြင်း
Terminal emulator (ဥပမာ- GNOME Terminal) ကို ဖွင့်လိုက်ပါက **Command Prompt** ကို မြင်တွေ့ရမည်ဖြစ်သည်။ ၎င်းသည် များသောအားဖြင့် အောက်ပါအတိုင်း ပေါ်လာတတ်သည်-

`username@hostname:current_directory$ `

* **$** အမှတ်အသားသည် သာမန်အသုံးပြုသူ (regular user) ဖြစ်ကြောင်း ပြသည်။
* **#** အမှတ်အသားသည် အခွင့်အာဏာအပြည့်ရှိသော အသုံးပြုသူ (root user/administrator) ဖြစ်ကြောင်း ပြသည်။

---

### Command Syntax (Command တည်ဆောက်ပုံ)
Linux command များသည် များသောအားဖြင့် အောက်ပါ pattern အတိုင်း အလုပ်လုပ်သည်-

`command [options] [arguments]`

1.  **Command:** သင်လုပ်ဆောင်လိုသော program အမည် (ဥပမာ- `ls`)။
2.  **Options:** Command ၏ အလုပ်လုပ်ပုံကို ပြောင်းလဲပေးသော ခလုတ်များ။ ၎င်းတို့ကို `-` (short options) သို့မဟုတ် `--` (long options) ဖြင့် စတင်လေ့ရှိသည် (ဥပမာ- `ls -l`)။
3.  **Arguments:** Command က အလုပ်လုပ်ရမည့် ပစ်မှတ် (target) (ဥပမာ- `ls /etc` တွင် `/etc` သည် argument ဖြစ်သည်)။

---

### အခြေခံ Command များ (Basic Commands)
* **`echo`:** သင်ရိုက်လိုက်သော စာသားကို မျက်နှာပြင်ပေါ်တွင် ပြန်ပြရန် သုံးသည်။
    * ဥပမာ- `echo Hello World`
* **`ls`:** လက်ရှိ folder ထဲရှိ file များနှင့် folder များကို စာရင်းပြုစုပြရန် သုံးသည်။
* **`pwd`:** သင်လက်ရှိ ရောက်ရှိနေသော လမ်းကြောင်း (Print Working Directory) ကို ပြရန် သုံးသည်။

---

### Command Types (Command အမျိုးအစားများ)
Linux Shell တွင် သင်ရိုက်ထည့်လိုက်သော command များသည် ၎င်းတို့၏ ဖြစ်တည်မှုအပေါ် မူတည်၍ အမျိုးအစား ၄ မျိုး ကွဲပြားနိုင်သည် -

1. **Internal Commands (Shell Built-ins):** ဤ command များသည် Shell (Bash) ၏ အတွင်းပိုင်းတွင် တစ်ပါတည်း ပါဝင်ပြီးသား ဖြစ်သည်။ ၎င်းတို့ကို run ရန် အခြားပြင်ပ file များ ရှာဖွေနေရန် မလိုသောကြောင့် အလွန်မြန်ဆန်သည် (ဥပမာ- `cd`, `exit`, `echo`)။
2. **External Commands:** ဤ command များသည် computer ၏ hard drive ပေါ်ရှိ directory တစ်ခုခုတွင် file အနေဖြင့် သီးခြားရှိနေသော program များ ဖြစ်သည်။ ၎င်းတို့ကို run သောအခါ Shell သည် `PATH` variable ထဲရှိ လမ်းကြောင်းများတွင် လိုက်လံရှာဖွေရသည် (ဥပမာ- `ls`, `cp`, `vi`)။
3. **Aliases:** ၎င်းတို့သည် ရှည်လျားသော command များကို အလွယ်တကူ ခေါ်သုံးနိုင်ရန် အသုံးပြုသူက သတ်မှတ်ထားသော "အမည်ပွား" များ ဖြစ်သည်။ ဥပမာ- `ls -l` ကို `ll` ဟု အမည်ပေးထားခြင်းမျိုး ဖြစ်သည်။
4. **Functions:** Shell အတွင်းတွင် ရေးသားထားသော script အသေးစားများ ဖြစ်ပြီး command တစ်ခုကဲ့သို့ ပြန်လည်ခေါ်ယူ အသုံးပြုနိုင်သည်။

#### The `type` Command (Command အမျိုးအစားကို စစ်ဆေးခြင်း)
သင်အသုံးပြုနေသော command တစ်ခုသည် မည်သည့်အမျိုးအစား ဖြစ်သည်ကို သိရှိရန် `type` command ကို အသုံးပြုနိုင်သည် -

* **ဥပမာ ၁ (Built-in):**
    `$ type cd`
    *Output:* `cd is a shell builtin`
* **ဥပမာ ၂ (External):**
    `$ type ls`
    *[span_0](start_span)Output:* `ls is /bin/ls` (၎င်းရှိသော လမ်းကြောင်းကိုပါ ပြပေးမည်)[span_0](end_span)
* **ဥပမာ ၃ (Alias):**
    `$ type ll`
    *Output:* `ll is aliased to 'ls -l'`

#### Finding External Commands (`which` နှင့် `whereis`)
External command များ၏ တည်နေရာကို သိရှိရန် အောက်ပါတို့ကို သုံးနိုင်သည် -

* **`which`:** Shell က လက်ရှိ အသုံးပြုမည့် executable file ၏ လမ်းကြောင်းကိုသာ ပြသည်။
    * ဥပမာ- `which python`
* **`whereis`:** Command ၏ binary file သာမက ၎င်းနှင့် သက်ဆိုင်သော source code နှင့် manual (man pages) များရှိရာ လမ်းကြောင်းများကိုပါ စုပေါင်းပြပေးသည်။

---

### Variables (ကိန်းရှင်များ)
Linux Shell သည် အချက်အလက်များကို သိမ်းဆည်းရန် Variables များကို အသုံးပြုသည်။

#### 1. Environment Variables
System တစ်ခုလုံးနှင့် သက်ဆိုင်သော setting များဖြစ်သည်။ ၎င်းတို့ကို အမြဲတမ်း စာလုံးအကြီးဖြင့် ရေးလေ့ရှိသည်။
* **`PATH`:** Executable file များ ရှာဖွေရမည့် နေရာများကို သိမ်းထားသည်။
* **`HOME`:** အသုံးပြုသူ၏ home directory လမ်းကြောင်း။
* **`USER`:** လက်ရှိ login ဝင်ထားသော အသုံးပြုသူအမည်။

Variable တစ်ခု၏ တန်ဖိုးကို ကြည့်လိုလျှင် ရှေ့တွင် **`$`** ထည့်၍ `echo` နှင့် တွဲသုံးရသည် (ဥပမာ- `echo $USER`)။

#### 2. Local Variables
အသုံးပြုသူမှ သတ်မှတ်ထားသော variable များဖြစ်သည်။
* သတ်မှတ်ပုံ- `MYVAR="Hello"`
* ပြန်ကြည့်ပုံ- `echo $MYVAR`

---

### Quoting (ကိုးကားခြင်း)
Shell တွင် အထူးသင်္ကေတများ (ဥပမာ- space, $, *) ပါဝင်သော စာသားများကို ကိုင်တွယ်ရန် quote များကို သုံးသည်-
* **Double Quotes (")**: Variable များကို အလုပ်လုပ်ခွင့်ပေးသော်လည်း space များကို စာသားတစ်ခုတည်းအဖြစ် သတ်မှတ်သည်။
* **Single Quotes (')**: အတွင်းရှိ အရာအားလုံးကို ရိုးရိုးစာသား (literal text) အဖြစ်သာ သတ်မှတ်သည်။ Variable များ အလုပ်မလုပ်ပါ။
* **Backslash (\)**: နောက်ကလိုက်လာသော စာလုံးတစ်လုံးတည်းကို အထူးသင်္ကေတအဖြစ်မှ ရိုးရိုးစာသားအဖြစ် ပြောင်းလဲပေးသည်။

---

### Command History
သင် ယခင်က ရိုက်ခဲ့သော command များကို ပြန်ကြည့်ရန်နှင့် ပြန်သုံးရန် ဖြစ်သည်။
* **`history` command:** ယခင်ရိုက်ခဲ့သမျှ command စာရင်းကို ပြသည်။
* **Up/Down Arrow keys:** Command တစ်ခုချင်းစီကို အပေါ်အောက် ရှာဖွေနိုင်သည်။
* **`!!`:** နောက်ဆုံးရိုက်ခဲ့သော command ကို ပြန် run သည်။
* **`CTRL + R`:** ယခင်ရိုက်ခဲ့သော command များကို ရှာဖွေရန် (Reverse search)။

---

### အကျဉ်းချုပ် (Summary)
ဤသင်ခန်းစာတွင် သင်ယူခဲ့ရသည်များမှာ-
* Shell နှင့် Bash ၏ အခြေခံသဘောတရား။
* Command များ၊ Options များနှင့် Arguments များ၏ တည်ဆောက်ပုံ။
* Command အမျိုးအစားများ အကြောင်း။
* Variables များကို အသုံးပြု၍ အချက်အလက် သိမ်းဆည်းပုံ။
* Quoting နည်းလမ်းများဖြင့် စာသားများကို ထိန်းချုပ်ပုံ။
* History command ကို သုံး၍ အလုပ်တွင်ကျယ်အောင် လုပ်ဆောင်ပုံ။
