### နိဒါန်း (Introduction)
ဤသင်ခန်းစာသည် Linux ပေါ်တွင် Command Line အသုံးပြု၍ ဖိုင်များနှင့် လမ်းညွှန် (Directory) များကို စီမံခန့်ခွဲပုံအကြောင်း ဖြစ်သည်။ 

ဖိုင် (File) ဆိုသည်မှာ အမည်တစ်ခုနှင့် သတ်မှတ်ချက် (Attributes) များရှိသော အချက်အလက်များ စုစည်းမှုဖြစ်သည်။  ဥပမာအားဖြင့် ဖုန်းထဲမှ ဓာတ်ပုံများကို ကွန်ပျူတာထဲသို့ ထည့်သွင်းပြီး အမည်ပေးလိုက်ပါက သင်၏ကွန်ပျူတာပေါ်တွင် ရုပ်ပုံဖိုင်များ ရရှိလာမည်ဖြစ်သည်။  ဤဖိုင်များတွင် နောက်ဆုံးဝင်ရောက်ကြည့်ရှုချိန် သို့မဟုတ် ပြင်ဆင်ချိန် စသည့် သတ်မှတ်ချက်များ ပါရှိသည်။ 

လမ်းညွှန် (Directory) ဆိုသည်မှာ ဖိုင်များကို စနစ်တကျ စုစည်းရန် အသုံးပြုသည့် အထူးဖိုင်အမျိုးအစား တစ်ခုဖြစ်သည်။  လမ်းညွှန်များကို ဖိုင်တွဲ (Folder) များကဲ့သို့ မှတ်ယူနိုင်သည်။  စာရွက်စာတမ်းဖိုင်တွဲများနှင့် မတူသည့်အချက်မှာ လမ်းညွှန်တစ်ခုအတွင်း အခြားလမ်းညွှန်များကို ထပ်ဆင့်ထည့်သွင်းနိုင်ခြင်း ဖြစ်သည်။ 

Linux စနစ်တွင် ဖိုင်များကို စီမံခန့်ခွဲရန် Command Line သည် အထိရောက်ဆုံး နည်းလမ်းဖြစ်သည်။  Shell နှင့် Command Line Tool များတွင် ပါဝင်သော လုပ်ဆောင်ချက်များသည် Graphical File Manager ကို အသုံးပြုခြင်းထက် ပိုမိုမြန်ဆန်ပြီး လွယ်ကူစေသည်။  ဤအပိုင်းတွင် သင်သည် ဖိုင်နှင့် လမ်းညွှန်များကို စီမံရန် `ls`, `mv`, `cp`, `pwd`, `find`, `touch`, `rm`, `rmdir`, `echo`, `cat` နှင့် `mkdir` စသည့် Command များကို အသုံးပြုရမည်ဖြစ်သည်။ 

### စာလုံးအကြီးအသေး ကွဲပြားမှု (Case Sensitivity)
Microsoft Windows နှင့်မတူဘဲ Linux စနစ်ရှိ ဖိုင်နှင့် လမ်းညွှန်အမည်များသည် စာလုံးအကြီးအသေး (Case Sensitive) ကွဲပြားသည်။  ဆိုလိုသည်မှာ `/etc/` နှင့် `/ETC/` သည် မတူညီသော လမ်းညွှန်နှစ်ခု ဖြစ်သည်။ 

အောက်ပါ Command များကို စမ်းသပ်ကြည့်ပါ- 
```bash
$ cd /
$ ls
bin dev home lib64 mnt proc run srv tmp var
boot etc lib media opt root sbin sys usr
$ cd ETC
bash: cd: ETC: No such file or directory
$ pwd
/
$ cd etc
$ pwd
/etc
```

`pwd` Command သည် သင်လက်ရှိရောက်ရှိနေသော လမ်းညွှန်ကို ဖော်ပြပေးသည်။ အထက်ပါ ဥပမာတွင် `/ETC` အမည်ဖြင့် လမ်းညွှန်မရှိသောကြောင့် ပြောင်းလဲ၍ မရသော်လည်း `/etc` လမ်းညွှန်သို့ ပြောင်းလဲခြင်းမှာ အောင်မြင်သည်ကို တွေ့ရမည်ဖြစ်သည်။

### လမ်းညွှန်များ ဖန်တီးခြင်း (Creating Directories)

`mkdir` Command ကို လမ်းညွှန်များ ဖန်တီးရန် အသုံးပြုသည်။ မိမိ၏ Home Directory အတွင်း လမ်းညွှန်အသစ်တစ်ခု ဖန်တီးကြည့်ပါ-
```bash
$ cd ~
$ pwd
/home/emma
$ ls
Desktop Documents Downloads
$ mkdir linux_essentials-2.4
$ ls
Desktop Documents Downloads linux_essentials-2.4
$ cd linux_essentials-2.4
$ pwd
/home/emma/linux_essentials-2.4
```

### ဖိုင်များ ဖန်တီးခြင်း (Creating Files)

`touch` command သည် ဖိုင်အသစ်တစ်ခု ဖန်တီးရန် အလွယ်ကူဆုံးနည်းလမ်း ဖြစ်သည်။ ၎င်းကို အဓိကအားဖြင့် ဖိုင်တစ်ခု၏ "နောက်ဆုံး ပြင်ဆင်ချိန်" (Timestamp) ကို အပ်ဒိတ်လုပ်ရန် အသုံးပြုသော်လည်း၊ ဖိုင်မရှိသေးပါက အကြောင်းအရာမပါဝင်သော (Empty file) ဖိုင်အသစ်တစ်ခုကို ဖန်တီးပေးသည်။

```bash
$ touch file1
$ ls -l file1
-rw-r--r-- 1 emma emma 0 Mar 5 10:20 file1
```

ဖိုင်ထဲသို့ စာသားများ ထည့်သွင်း၍ ဖန်တီးလိုပါက echo command နှင့် redirection operator (`>`) ကို အသုံးပြုနိုင်သည်။
```bash
$ echo "Hello Linux" > file2
$ cat file2
Hello Linux
```

### ဖိုင်အမည် ပြောင်းလဲခြင်း (Renaming Files)
Linux တွင် ဖိုင်အမည်ပြောင်းရန် သီးသန့် command မရှိဘဲ `mv` (move) command ကိုပင် အသုံးပြုရသည်။ ဖိုင်တစ်ခုကို အမည်သစ်ဖြင့် နေရာဟောင်းမှာပင် ပြန်သိမ်းခြင်းသည် အမည်ပြောင်းလဲခြင်းပင် ဖြစ်သည်။
```bash
$ mv file1 file_new
$ ls
file_new file2
```

### ဖိုင်များ ရွှေ့ပြောင်းခြင်း (Moving Files)
ဖိုင်တစ်ခု (သို့မဟုတ်) တစ်ခုထက်ပိုသော ဖိုင်များကို အခြား လမ်းညွှန် (Directory) တစ်ခုသို့ ရွှေ့ပြောင်းရန် `mv` command ကို အသုံးပြုသည်။
```bash
$ mkdir backup
$ mv file_new file2 backup/
$ ls backup/
file_new file2
```

### File များနှင့် Directory များကို ဖျက်ပစ်ခြင်း (Deleting Files and Directories)
ဖိုင်များကို ဖျက်ပစ်ရန် `rm` (remove) command ကို အသုံးပြုသည်။ သတိပြုရန်- Command line မှ ဖျက်လိုက်သော ဖိုင်များသည် အမှိုက်ပုံး (Trash/Recycle Bin) သို့ ရောက်မသွားဘဲ အပြီးတိုင် ပျက်စီးသွားမည် ဖြစ်သည်။
* File တစ်ခုကို ဖျက်ရန်: `rm filename` 
* Directory အလွတ်တစ်ခုကို ဖျက်ရန်: `rmdir directory_name`
* အထဲတွင် ဖိုင်များရှိနေသော Directory ကို ဖျက်ရန်: `rm -r directory_name` (-r သည် recursive ကို ဆိုလိုသည်)
```bash
$ rm backup/file2
$ rm -r backup/
```

### File များနှင့် Directory များကို ကူးယူခြင်း (Copying Files and Directories)
ဖိုင်များကို မိတ္တူကူးရန် `cp` (copy) command ကို အသုံးပြုသည်။
* File တစ်ခုကို ကူးယူရန်: `cp source_file destination_file`
* Directory တစ်ခုလုံးကို ကူးယူရန်: `cp -r source_dir destination_dir`
```bash
$ cp file_new file_copy
$ cp -r project_dir project_backup
```

### Globbing (Wildcards အသုံးပြုခြင်း)
Shell သည် ဖိုင်အမည် အများအပြားကို တစ်ပြိုင်နက် ရွေးချယ်နိုင်ရန် "Wildcards" သို့မဟုတ် "Globbing" ကို အသုံးပြုခွင့်ပေးသည်။ `touch file{1..7} file{A..C} file{10..12}`
* `*` (Asterisk): စာလုံးအရေအတွက် မည်မျှမဆို (လုံးဝမပါဝင်ခြင်း အပါအဝင်) ကို ကိုယ်စားပြုသည်။
  * `ls *.txt` (အဆုံးတွင် .txt ပါသော ဖိုင်အားလုံးကို ပြပေးမည်)
* `**` (Double Asterisk): ပိုမိုစွမ်းအားထက်မြက်ပြီး Recursive (အဆင့်ဆင့်ဆင်းသွားသော) wildcard တစ်ခု ဖြစ်ပါတယ်။
  * `ls **/*.txt` (လက်ရှိ directory ထဲမှာရှိတဲ့ ဖိုင်တွေ၊ Folder တွေသာမက ၎င်းရဲ့ အောက်ခြေ Subdirectories အားလုံးထဲမှာရှိတဲ့ အဆုံးတွင် `.txt` ပါသော ဖိုင်အားလုံးကို ပြပေးမည်)
* `?` (Question Mark): စာလုံး တစ်လုံးတည်း (Single character) ကိုသာ ကိုယ်စားပြုသည်။
  * `ls file?` (file1, file2, fileA စသည်တို့ကို ပြပေးမည်)
* `[]` (Square Brackets): ကွင်းအတွင်း ပါဝင်သော စာလုံးများထဲမှ တစ်လုံးကို ကိုယ်စားပြုသည်။
  * `ls file[123]` (file1, file2, file3 ကိုသာ ပြပေးမည်)
  * `ls file[3-5]` (file3, file4, file5 ကိုသာ ပြပေးမည်)
  * `ls file[12][12]` (file11, file12 ကိုသာ ပြပေးမည်)
* `[!]` (Logical NOT): ကွင်းအတွင်း ပါဝင်သော စာလုံးများထဲမှ မပါဝင်သည်ကို ကိုယ်စားပြုသည်။
  * `ls file[!123]` (file4, file5, file6, file7, fileA, fileB, fileC ကိုသာ ပြပေးမည်)
* `{}` (Curly Brackets): ကွင်းအတွင်း ပါဝင်သော စာလုံးများအားလုံးကို ကိုယ်စားပြုသည်။
  * `ls file{1,2}` (file1, file2 ကိုသာ ပြပေးမည်)
  * `ls file{4..6}` (file4, file5 နှင့် file6 ကိုသာ ပြပေးမည်)

#### Character Classes (အက္ခရာ အမျိုးအစားများ)

rackets များအတွင်း အောက်ပါ character classes များကို အသုံးပြု၍ သီးခြားအမျိုးအစားဝင် စာလုံးများကို ရှာဖွေနိုင်သည်-

- **`[:digit:]`**: ကိန်းဂဏန်းများ (၀ မှ ၉)
- **`[:alpha:]`**: အက္ခရာများ (A-Z, a-z)
- **`[:alnum:]`**: အက္ခရာနှင့် ကိန်းဂဏန်းများ ပေါင်းစပ်မှု
- **`[:upper:]`**: အက္ခရာ အကြီးများ
- **`[:lower:]`**: အက္ခရာ အသေးများ
- **`[:blank:]`**: Space နှင့် Tab character များ (Horizontal Whitespace)
- **`[:space:]`**: Space, Tab character, New Line, စသည်ဖြင့် (Any Whitespace)

**`[:classname:]`** ဟု ဖော်ပြထားခြင်းမှာ အထက်ပါ class အမည်များကို bracket များအတွင်း ထည့်သွင်းအသုံးပြုရသည့် ပုံစံကို ဆိုလိုခြင်းဖြစ်သည်။ ဥပမာအားဖြင့် ကိန်းဂဏန်းတစ်ခုခု ပါဝင်သော ဖိုင်များကို ရှာလိုပါက `[[:digit:]]` ဟု ရေးသားရမည်။ အပြင်ဘက် bracket သည် wildcard အဖြစ် ဆောင်ရွက်ပြီး အတွင်းဘက်ရှိ `[:digit:]` သည် character class ကို ကိုယ်စားပြုသည်။

ဥပမာ-
```bash
$ ls file[[:digit:]]
file1	file2	file3	file4	file5	file6	file7
$ ls file[[:digit:]A]
file1	file2	file3	file4	file5	file6	file7 fileA
$ ls file[[:digit:]][[:digit:]]
file10	file11	file12
```

### အနှစ်ချုပ် (Summary)

ဤသင်ခန်းစာတွင် Linux ဖိုင်စနစ် (File System) အတွင်း အခြေခံကျသော လုပ်ဆောင်ချက်များကို လေ့လာခဲ့ပြီး ဖြစ်သည်။ ဖိုင်နှင့် လမ်းညွှန်များကို ဖန်တီးခြင်း၊ အမည်ပြောင်းခြင်း၊ နေရာရွှေ့ပြောင်းခြင်းနှင့် ဖျက်ပစ်ခြင်းတို့သည် Linux စနစ် စီမံခန့်ခွဲသူတစ်ဦးအတွက် မရှိမဖြစ် ကျွမ်းကျင်ရမည့် အရည်အချင်းများ ဖြစ်သည်။

မှတ်သားရန် အချက်များ-
* Linux သည် **Case Sensitive** ဖြစ်သောကြောင့် `File.txt` နှင့် `file.txt` သည် မတူညီပါ။
* `mv` command ကို ဖိုင်ရွှေ့ရန်သာမက အမည်ပြောင်းရန်ပါ အသုံးပြုသည်။
* `rm` command ဖြင့် ဖျက်လိုက်သော ဖိုင်များကို ပြန်လည်ရယူရန် ခဲယဉ်းသဖြင့် သတိပြုသုံးစွဲရမည်။
* **Globbing** (`*`, `?`, `[]`, `{}`) ကို အသုံးပြုခြင်းဖြင့် အလုပ်တွင်ကျယ်မှုကို များစွာ အထောက်အကူပြုစေသည်။
