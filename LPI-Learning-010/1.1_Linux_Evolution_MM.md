### နိဒါန်း (Introduction)
Linux သည် ကမ္ဘာပေါ်တွင် အသုံးအများဆုံး Operating Systems (OS) များထဲမှ တစ်ခုဖြစ်သည်။ ၎င်း၏ ဖွံ့ဖြိုးတိုးတက်မှုကို ၁၉၉၁ ခုနှစ်တွင် **Linus Torvalds** မှ စတင်ခဲ့သည်။ Linux သည် ၁၉၇၀ ပြည့်နှစ်များအတွင်း AT&T Laboratories မှ ထုတ်လုပ်ခဲ့သော Unix အမည်ရှိ အခြား OS တစ်ခုကို အခြေခံ၍ စိတ်ကူးရရှိခဲ့ခြင်းဖြစ်သည်။ ထိုခေတ်အခါက Unix ကို ကွန်ပျူတာ အသေးစားများအတွက် ရည်ရွယ်ခဲ့သည်။ ထိုအချိန်က "အသေးစား" ဟု ဆိုရာတွင် လေအေးပေးစက် တပ်ဆင်ထားသော ခန်းမတစ်ခုလုံးစာ နေရာယူရန် မလိုဘဲ ဒေါ်လာတစ်သန်းအောက် တန်ဖိုးရှိသည့် စက်များကို ဆိုလိုခြင်းဖြစ်သည်။

နောက်ပိုင်းတွင် လူနှစ်ဦး မနိုင်သည့် စက်များကိုပါ ဆိုလိုလာကြသည်။ ထိုစဉ်က ရုံးသုံး x86 platform များတွင် အသုံးပြုနိုင်သော သင့်တင့်သည့် Unix system များ မရှိသေးပေ။ ထို့ကြောင့် ကျောင်းသားတစ်ဦး ဖြစ်သော Linus သည် x86 platform ပေါ်တွင် အသုံးပြုနိုင်မည့် Unix နှင့် ဆင်တူသော OS တစ်ခုကို စတင် အကောင်အထည်ဖော်ခဲ့သည်။ Linux သည် Unix ၏ မူဝါဒများနှင့် အခြေခံ အယူအဆများကို အသုံးပြုသော်လည်း Linux ကိုယ်တိုင်တွင် Unix code များ ပါဝင်ခြင်း မရှိဘဲ သီးခြားလွတ်လပ်သော project တစ်ခု ဖြစ်သည်။ Linux ကို ကုမ္ပဏီတစ်ခုတည်းမှ ထောက်ပံ့ထားခြင်း မဟုတ်ဘဲ နိုင်ငံတကာမှ ပရိုဂရမ်မာများ စုပေါင်းထားသည့် အဖွဲ့အစည်း (community) မှ ထောက်ပံ့ထားခြင်း ဖြစ်သည်။ ၎င်းကို အခမဲ့ ရရှိနိုင်ပြီး မည်သူမဆို ကန့်သတ်ချက်မရှိ အသုံးပြုနိုင်သည်။

### Distributions
Linux distribution ဆိုသည်မှာ Linux kernel နှင့် အသုံးပြုသူ community သို့မဟုတ် ကုမ္ပဏီတစ်ခုမှ စုစည်းထိန်းသိမ်းထားသော application များ၏ အစုအဝေး (bundle) ဖြစ်သည်။ Distribution တစ်ခု၏ ရည်ရွယ်ချက်မှာ သတ်မှတ်ထားသော အသုံးပြုမှုပုံစံ သို့မဟုတ် အသုံးပြုသူအုပ်စုအတွက် OS ကို အကောင်းဆုံးဖြစ်အောင် ပြင်ဆင်ထားရန် ဖြစ်သည်။

Distribution များတွင် software installation နှင့် system administration အတွက် သီးသန့် tool များ ပါဝင်လေ့ရှိသည်။ ထို့ကြောင့် အချို့ distribution များကို desktop များတွင် အလွယ်တကူ အသုံးပြုနိုင်ရန် ရည်ရွယ်ပြီး၊ အချို့ကို server များတွင် resource များကို အထိရောက်ဆုံး အသုံးချနိုင်ရန် ရည်ရွယ်၍ အသုံးပြုကြသည်။

Distribution များကို ၎င်းတို့၏ မိသားစု (distribution family) အလိုက် ခွဲခြားနိုင်သည်။

#### Debian Family
* **Package Manager:** `dpkg` ကို အသုံးပြု၍ software များကို စီမံခန့်ခွဲသည်။
* **Package Format:** `.deb` format ကို အသုံးပြုသည်။
* **Debian GNU/Linux:** ၁၉၉၃ ခုနှစ်တွင် Ian Murdock မှ စတင်ခဲ့ပြီး အလွန်ခိုင်မာ စိတ်ချရသော OS ကို ပေးစွမ်းရန် ရည်ရွယ်သည်။ ၎င်းသည် proprietary software (မူပိုင်ခွင့်ရှိ software) များကို default အနေဖြင့် ထည့်သွင်းပေးထားခြင်း မရှိပါ။
* **Ubuntu:** ၂၀၀၄ ခုနှစ်တွင် Mark Shuttleworth မှ စတင်ခဲ့ပြီး အသုံးပြုရလွယ်ကူသော desktop environment ကို ပေးစွမ်းရန် ရည်ရွယ်သည်။ ၎င်းကို ၆ လတစ်ကြိမ် release အသစ်ထုတ်ဝေပြီး ၂ နှစ်တစ်ကြိမ် Long-Term Support (LTS) release ထုတ်ဝေသည်။

#### Red Hat Family
* **Package Format:** `.rpm` format ကို အသုံးပြုသည်။
* **Red Hat Enterprise Linux (RHEL):** ကုမ္ပဏီများအတွက် ခိုင်မာသော enterprise solution အဖြစ် ရည်ရွယ်သည်။ ၂၀၁၉ ခုနှစ်တွင် IBM မှ Red Hat ကို ဝယ်ယူခဲ့သည်။ အချို့ အစိတ်အပိုင်းများအတွက် အခပေး subscription လိုအပ်သည်။
* **CentOS:** RHEL ၏ free source code ကို အခြေခံ၍ ထုတ်လုပ်ထားပြီး လုံးဝ အခမဲ့ အသုံးပြုနိုင်သည်။ သို့သော် commercial support မရရှိနိုင်ပါ။
* **Fedora:** ၂၀၀၃ ခုနှစ်တွင် စတင်ခဲ့ပြီး desktop ကွန်ပျူတာများအတွက် ရည်ရွယ်သည်။ နည်းပညာသစ်များကို လျင်မြန်စွာ လက်ခံကျင့်သုံးပြီး RHEL အတွက် စမ်းသပ်ရာနေရာ (test-bed) အဖြစ်လည်း သတ်မှတ်ကြသည်။

#### SUSE Family
* **YaST Tool:** Software နှင့် hardware များ၊ server များနှင့် network များကို configure လုပ်ရန် အသုံးပြုသော နာမည်ကျော် tool ဖြစ်သည်။
* **openSUSE:** Developer များနှင့် အသုံးပြုသူများ စမ်းသပ်နိုင်ရန် ထုတ်လုပ်ထားသော အခမဲ့ distribution ဖြစ်သည်။

#### အခြား Distribution များ
* **QubesOS:** လုံခြုံရေး အလွန်ကောင်းမွန်သော desktop environment။
* **Kali Linux:** Penetration testers များ software အားနည်းချက်များကို ရှာဖွေရန် အသုံးပြုသည်။
* **Docker Containers:** သီးသန့် run ရန်အတွက် အလွန်သေးငယ်သော Linux distribution များလည်း ရှိသည်။

### Embedded Systems
Embedded systems ဆိုသည်မှာ ပိုမိုကြီးမားသော system တစ်ခုအတွင်း သီးသန့် လုပ်ဆောင်ချက်တစ်ခုအတွက် ဒီဇိုင်းထုတ်ထားသော hardware နှင့် software ပေါင်းစပ်မှု ဖြစ်သည်။ ၎င်းတို့ကို မော်တော်ကား၊ ဆေးဘက်ဆိုင်ရာနှင့် စစ်ဘက်ဆိုင်ရာ အသုံးအဆောင်များတွင် တွေ့ရသည်။ Linux ၏ အားသာချက်များမှာ license ခပေးရန် မလိုခြင်းနှင့် vendor တိုင်းတွင် အသုံးပြုနိုင်ခြင်းတို့ ဖြစ်သည်။

* **Android:** Google မှ တီထွင်ထားသော mobile OS ဖြစ်သည်။ Linux kernel ကို အခြေခံပြီး touchscreen device များ (TV နှင့် နာရီများ အပါအဝင်) အတွက် အဓိက အသုံးပြုသည်။ Android Open Source Project (AOSP) အနေဖြင့် အခမဲ့ ရရှိနိုင်သည်။
* **Raspberry Pi & Raspbian:** Raspberry Pi သည် ခရက်ဒစ်ကတ် အရွယ်အစားရှိ တန်ဖိုးနည်း ကွန်ပျူတာ ဖြစ်သည်။ ပရိုဂရမ်သင်ကြားရန်နှင့် hardware development များအတွက် အသုံးပြုသည်။ ၎င်းတွင် အသုံးပြုသော နာမည်ကျော် distribution မှာ **Raspbian** (Debian family) ဖြစ်သည်။

### Linux and the Cloud
Cloud computing ဆိုသည်မှာ computing resource များကို စံသတ်မှတ်ထားသော နည်းလမ်းဖြင့် အသုံးပြုခြင်း ဖြစ်သည်။ ၂၀၁၇ ခုနှစ် အစီရင်ခံစာများအရ Public Cloud လုပ်ဆောင်ချက်များ၏ ၉၀% သည် Linux ပေါ်တွင် run နေခြင်း ဖြစ်သည်။ Amazon Web Services (AWS), Google Cloud Platform (GCP) နှင့် Microsoft Azure တို့တွင် Linux-based virtual machines များကို ရရှိနိုင်သည်။

Linux ကို များသောအားဖြင့် **Infrastructure as a Service (IaaS)** အနေဖြင့် ပေးလေ့ရှိပြီး မိနစ်ပိုင်းအတွင်း virtual machines များကို တည်ဆောက်နိုင်သည်။

### အကျဉ်းချုပ် (Summary)
ဤသင်ခန်းစာတွင် သင်ယူခဲ့ရသည်များမှာ-
* Linux ၏ သမိုင်းကြောင်းနှင့် Unix နှင့် ဆက်နွယ်မှု။
* Linux distribution များ၏ အယူအဆနှင့် အဓိက distribution family များ (Debian, Red Hat, SUSE)။
* Embedded systems များတွင် Linux ၏ အခန်းကဏ္ဍ (Android, Raspberry Pi)။
* Cloud computing တွင် Linux ကို အသုံးချပုံ။
