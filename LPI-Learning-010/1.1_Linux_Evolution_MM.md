## 1.1 Lesson 1

### နိဒါန်း (Introduction)
Linux သည် ကမ္ဘာပေါ်တွင် လူကြိုက်အများဆုံး Operating Systems (OS) များထဲမှ တစ်ခုဖြစ်သည်။ [span_10](start_span)၎င်းကို ၁၉၉၁ ခုနှစ်တွင် Linus Torvalds မှ စတင်ဖန်တီးခဲ့သည်[span_10](end_span)။ [span_11](start_span)ဤ OS ကို ၁၉၇၀ ပြည့်လွန်နှစ်များက AT&T Laboratories မှ ထုတ်လုပ်ခဲ့သော Unix OS ကို အခြေခံ၍ စိတ်ကူးရရှိခဲ့ခြင်း ဖြစ်သည်[span_11](end_span)။ [span_12](start_span)ထိုစဉ်က Unix ကို ကွန်ပျူတာ အသေးစားများအတွက် ရည်ရွယ်ခဲ့သော်လည်း ထိုခေတ်က "အသေးစား" ဟူသည်မှာ လေအေးပေးစက်တပ်ဆင်ထားသော အခန်းတစ်ခန်းလုံးစာ မလိုအပ်ဘဲ ဒေါ်လာတစ်သန်းအောက်သာ ကုန်ကျသည့် စက်များကို ဆိုလိုခြင်းဖြစ်သည်[span_12](end_span)။

[span_13](start_span)နောက်ပိုင်းတွင် လူနှစ်ဦး မနိုင်သည့် စက်များကို အသေးစားဟု သတ်မှတ်ခဲ့ကြသော်လည်း ရုံးသုံးကွန်ပျူတာများ (x86 platform) တွင် သက်သာသော ဈေးနှုန်းဖြင့် အသုံးပြုနိုင်သည့် Unix system မရှိသေးပေ[span_13](end_span)။ [span_14](start_span)ထို့ကြောင့် ထိုစဉ်က ကျောင်းသားတစ်ဦးဖြစ်သည့် Linus သည် x86 platform ပေါ်တွင် အလုပ်လုပ်နိုင်မည့် Unix နှင့် ဆင်တူသော OS တစ်ခုကို စတင်အကောင်အထည်ဖော်ခဲ့ခြင်း ဖြစ်သည်[span_14](end_span)။ [span_15](start_span)Linux သည် Unix ၏ အခြေခံမူများနှင့် စိတ်ကူးစိတ်သန်းများကို အသုံးပြုထားသော်လည်း ၎င်းကိုယ်တိုင်သည် သီးခြားလွတ်လပ်သော project တစ်ခုဖြစ်ပြီး Unix code များ ပါဝင်ခြင်း မရှိပါ[span_15](end_span)။ [span_16](start_span)Linux သည် ကုမ္ပဏီတစ်ခုတည်းမှ ပိုင်ဆိုင်ခြင်း မဟုတ်ဘဲ နိုင်ငံတကာမှ Programmer များ စုပေါင်းထားသော Community မှ ထောက်ပံ့ထားခြင်း ဖြစ်သည်[span_16](end_span)။

### Distributions
[span_17](start_span)Linux Distribution (သို့မဟုတ် Distro) ဆိုသည်မှာ Linux Kernel နှင့် ကုမ္ပဏီ သို့မဟုတ် Community တစ်ခုမှ ထိန်းသိမ်းထားသော Application များစွာကို စုစည်းထားသည့် အစုအဝေး (Bundle) တစ်ခု ဖြစ်သည်[span_17](end_span)။ [span_18](start_span)Distribution တစ်ခု၏ ရည်ရွယ်ချက်မှာ အသုံးပြုသူ အုပ်စုတစ်ခု သို့မဟုတ် သတ်မှတ်ထားသော အသုံးပြုမှု ပုံစံတစ်ခုအတွက် Kernel နှင့် Application များကို အကောင်းဆုံးဖြစ်အောင် ပြင်ဆင်ပေးရန် ဖြစ်သည်[span_18](end_span)။

Distribution များကို ၎င်းတို့၏ မိသားစု (Family) အလိုက် အောက်ပါအတိုင်း ခွဲခြားနိုင်သည်-

#### 1. Debian Family
* **[span_19](start_span)Package Manager**: `dpkg` ကို အသုံးပြုပြီး software များကို စီမံသည်[span_19](end_span)။
* **[span_20](start_span)Package Format**: `.deb` format ကို အသုံးပြုသည်[span_20](end_span)။
* **[span_21](start_span)Debian GNU/Linux**: Ian Murdock မှ ၁၉၉၃ ခုနှစ်တွင် စတင်ခဲ့ပြီး အလွန်ခိုင်မာစိတ်ချရသော OS ဖြစ်ရန် ရည်ရွယ်သည်[span_21](end_span)။ [span_22](start_span)အသုံးပြုသူ၏ လွတ်လပ်ခွင့်ကို လေးစားသောကြောင့် Proprietary software (မူပိုင်ဆော့ဖ်ဝဲ) များကို မူလအတိုင်း ထည့်သွင်းထားခြင်း မရှိပါ[span_22](end_span)။
* **[span_23](start_span)Ubuntu**: Mark Shuttleworth မှ ၂၀၀၄ ခုနှစ်တွင် စတင်ခဲ့ပြီး အသုံးပြုရလွယ်ကူသော Desktop environment တစ်ခုဖြစ်ရန် ရည်ရွယ်သည်[span_23](end_span)။ [span_24](start_span)၆ လတစ်ကြိမ် Version အသစ်ထုတ်ဝေပြီး ၂ နှစ်တစ်ကြိမ် Long-term support (LTS) version ကို ထုတ်ဝေသည်[span_24](end_span)။

#### 2. Red Hat Family
* **[span_25](start_span)Package Format**: `.rpm` format ကို အသုံးပြုသည်[span_25](end_span)။
* **[span_26](start_span)Red Hat Enterprise Linux (RHEL)**: ၁၉၉၄ ခုနှစ်တွင် စတင်ခဲ့ပြီး စီးပွားရေးလုပ်ငန်းသုံး server များအတွက် စိတ်ချရသော solution အဖြစ် ထုတ်လုပ်ထားသည်[span_26](end_span)။ [span_27](start_span)၎င်းကို အသုံးပြုရန် ဝန်ဆောင်ခ (Subscription) ပေးဆောင်ရသည်[span_27](end_span)။
* **[span_28](start_span)CentOS**: RHEL ၏ free ရရှိနိုင်သော source code များကို အခြေခံ၍ ပြန်လည်ထုတ်လုပ်ထားသော distribution ဖြစ်သည်[span_28](end_span)။ [span_29](start_span)အခမဲ့ဖြစ်သော်လည်း Commercial support မရရှိနိုင်ပါ[span_29](end_span)။
* **[span_30](start_span)Fedora**: Red Hat မှ ထိန်းသိမ်းထားပြီး Desktop များအတွက် ရည်ရွယ်သည်[span_30](end_span)။ [span_31](start_span)နည်းပညာအသစ်များကို လျင်မြန်စွာ စမ်းသပ်အသုံးပြုလေ့ရှိပြီး RHEL အတွက် စမ်းသပ်ကွင်းသဖွယ် ဖြစ်သည်[span_31](end_span)။

#### 3. SUSE Family
* **[span_32](start_span)SUSE Linux**: ၁၉၉၂ ခုနှစ်တွင် ဂျာမနီနိုင်ငံ၌ စတင်ခဲ့သည်[span_32](end_span)။ **[span_33](start_span)YaST** ဟုခေါ်သော hardware နှင့် software များကို install လုပ်ရန်နှင့် configure လုပ်ရန် အသုံးပြုသည့် tool ကြောင့် လူသိများ သည်[span_33](end_span)။
* **[span_34](start_span)openSUSE**: Developer များနှင့် အသုံးပြုသူများ စနစ်ကို စမ်းသပ်ရန်နှင့် ပိုမိုဖွံ့ဖြိုးအောင် လုပ်ဆောင်နိုင်ရန်အတွက် ၂၀၀၄ ခုနှစ်တွင် အခမဲ့ထုတ်ဝေခဲ့သည်[span_34](end_span)။

#### 4. အခြား သီးခြား Distribution များ
* **[span_35](start_span)QubesOS**: အလွန်လုံခြုံစိတ်ချရသော Desktop environment အတွက် ထုတ်လုပ်ထားသည်[span_35](end_span)။
* **[span_36](start_span)Kali Linux**: Penetration testers များအတွက် software များ၏ အားနည်းချက်ကို စမ်းသပ်ရန် (Exploiting) ထုတ်လုပ်ထားသည်[span_36](end_span)။

### Embedded Systems
[span_37](start_span)Embedded systems ဆိုသည်မှာ ပိုမိုကြီးမားသော System တစ်ခုအတွင်း သတ်မှတ်ထားသော လုပ်ဆောင်ချက်တစ်ခုတည်းကို လုပ်ဆောင်ရန်အတွက် Hardware နှင့် Software ပေါင်းစပ်ဖန်တီးထားသော စနစ်များ ဖြစ်သည်[span_37](end_span)။ ၎င်းတို့ကို မော်တော်ယာဉ်၊ [span_38](start_span)ဆေးဘက်ဆိုင်ရာနှင့် စစ်ဘက်ဆိုင်ရာ အသုံးချပစ္စည်းများတွင် တွေ့ရလေ့ရှိသည်[span_38](end_span)။

* **[span_39](start_span)Android**: Linux Kernel ပေါ်တွင် အခြေခံထားသော လူသုံးအများဆုံး mobile OS တစ်ခု ဖြစ်သည်[span_39](end_span)။
* **[span_40](start_span)Raspberry Pi**: စျေးနှုန်းသက်သာသော credit card အရွယ်အစားရှိ ကွန်ပျူတာဖြစ်ပြီး ပညာရေးနှင့် embedded project များတွင် အသုံးများသည်[span_40](end_span)။ [span_41](start_span)၎င်းအတွက် အဓိက OS မှာ **Raspbian** ဖြစ်သည်[span_41](end_span)။

### Linux and the Cloud
[span_42](start_span)Cloud Computing ဆိုသည်မှာ ကွန်ပျူတာ အရင်းအမြစ်များကို Internet မှတစ်ဆင့် လိုအပ်သလို အသုံးပြုနိုင်ခြင်းကို ဆိုလိုသည်[span_42](end_span)။ [span_43](start_span)Linux သည် Cloud နည်းပညာ၏ အဓိက အစိတ်အပိုင်းဖြစ်ပြီး Amazon Web Services (AWS) သို့မဟုတ် Google Cloud Platform (GCP) ကဲ့သို့သော Cloud provider အများစုသည် Linux ကို အခြေခံ၍ အလုပ်လုပ်ကြသည်[span_43](end_span)။
