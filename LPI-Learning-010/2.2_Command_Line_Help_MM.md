### နိဒါန်း (Introduction)
Linux system တွင် command ပေါင်း ထောင်နှင့်ချီ၍ ရှိပါသည်။ ၎င်းတို့အားလုံးကို အလွတ်မှတ်ရန် မဖြစ်နိုင်သကဲ့သို့ မှတ်ရန်လည်း မလိုအပ်ပါ။ Linux တွင် command တစ်ခုစီ၏ အလုပ်လုပ်ပုံ၊ အသုံးပြုနိုင်သော option များနှင့် ဥပမာများကို system အတွင်း၌ပင် ပြည့်ပြည့်စုံစုံ ထည့်သွင်းပေးထားပါသည်။ ဤသင်ခန်းစာတွင် ထိုအကူအညီများကို မည်သို့ ရှာဖွေရမည်ကို လေ့လာပါမည်။

### Getting Help on the Command Line (Command Line ပေါ်တွင် အကူအညီရယူခြင်း)

#### 1. The `--help` Option
Command အများစုတွင် ၎င်းတို့၏ အခြေခံ အသုံးပြုပုံကို အကျဉ်းချုပ် ပြသပေးသည့် `--help` သို့မဟုတ် `-h` option ပါရှိပါသည်။
* **ဥပမာ:** `$ ls --help`
၎င်းသည် command ၏ syntax (တည်ဆောက်ပုံ) နှင့် အသုံးများသော option များကို screen ပေါ်တွင် အမြန်ပြသပေးမည် ဖြစ်သည်။


#### 2. Manual Pages (`man`)
`man` command သည် Linux တွင် အခြေခံအကျဆုံးနှင့် အသုံးအဝင်ဆုံး help system ဖြစ်သည်။ ၎င်းသည် command တစ်ခု၏ ပြည့်စုံသော လက်စွဲစာအုပ် (Manual) ကို ပြသပေးသည်။
* **အသုံးပြုပုံ:** `$ man <command_name>` (ဥပမာ- `$ man ls`)
* **Manual အတွင်း သွားလာပုံ:**
    * **Space bar:** နောက်တစ်မျက်နှာသို့ သွားရန်။
    * **Enter:** တစ်ကြောင်းချင်းစီ ဆင်းရန်။
    * **`q`:** Manual မှ ပြန်ထွက်ရန်။
    * **`/keyword`:** စာသားများကို ရှာဖွေရန်။

Manual စာမျက်နှာများကို အပိုင်း (Section) များ ခွဲခြားထားပါသည်:
* **Section 1:** User Commands (သာမန် command များ)
* **Section 2:** System Calls
* **Section 5:** File Formats (configuration file များအကြောင်း)
* **Section 8:** System Administration commands (root သာ သုံးနိုင်သော command များ)

#### 3. The `info` Command
`info` သည် `man` ထက် ပိုမိုပြည့်စုံပြီး စာအုပ်တစ်အုပ်ကဲ့သို့ link များဖြင့် ချိတ်ဆက်ထားသော help system ဖြစ်သည်။ ၎င်းသည် GNU project မှ command များအတွက် ပိုမိုအသေးစိတ်ကျသော documentation များကို ပေးသည်။
* **အသုံးပြုပုံ:** `$ info <command_name>`

---

### Finding Documentation in `/usr/share/doc/`
အချို့သော software များသည် ၎င်းတို့၏ အသေးစိတ် documentation များကို system အတွင်းရှိ `/usr/share/doc/` directory အောက်တွင် သိမ်းဆည်းထားလေ့ရှိသည်။ ထိုနေရာတွင် README file များ၊ ဥပမာ configuration များ နှင့် အခြားသော လမ်းညွှန်ချက်များကို file အနေဖြင့် ဖတ်ရှုနိုင်ပါသည်။

---

### Locating Files (File များ ရှာဖွေခြင်း)
Command တစ်ခု သို့မဟုတ် file တစ်ခု မည်သည့်နေရာတွင် ရှိသည်ကို ရှာဖွေရန် အောက်ပါ command များကို သုံးနိုင်သည်-

* **`which`:** Command တစ်ခု၏ executable file (program file) မည်သည့်နေရာတွင် ရှိသည်ကို ပြသည်။
    * **ဥပမာ:** `$ which ls` -> `/bin/ls`
* **`whereis`:** Command ၏ binary file၊ source code နှင့် manual page တို့၏ တည်နေရာအားလုံးကို ပြသည်။
    * **ဥပမာ:** `$ whereis ls`
* **`locate`:** System အတွင်းရှိ မည်သည့် file ကိုမဆို အမည်ဖြင့် အမြန်ရှာဖွေပေးသည်။ ၎င်းသည် database ကို အသုံးပြုသောကြောင့် အလွန်မြန်သော်လည်း database အသစ်မဖြစ်သေးလျှင် နောက်ဆုံးထည့်ထားသော file ကို တွေ့မည်မဟုတ်ပါ။
    * **ဥပမာ:** `$ locate crontab`
    * **မှတ်ချက်:** Database ကို update လုပ်ရန် `sudo updatedb` command ကို သုံးရသည်။

### The `find` Command
`find` command သည် directory တစ်ခုအတွင်း (၎င်း၏ subdirectory များအပါအဝင်) file များကို သတ်မှတ်ထားသော အချက်အလက်များပေါ် မူတည်၍ ရှာဖွေရန် အသုံးပြုသည်။ ၎င်းသည် `locate` ကဲ့သို့ database ကို မသုံးဘဲ disk ပေါ်တွင် တိုက်ရိုက်ရှာဖွေသောကြောင့် ပိုမိုတိကျသော်လည်း အနည်းငယ် ပိုကြာနိုင်သည်။

#### Basic Syntax (အခြေခံ တည်ဆောက်ပုံ)
`find [starting_directory] [options] [expression]`

#### အသုံးများသော Options များနှင့် ဥပမာများ
* **အမည်ဖြင့် ရှာဖွေခြင်း (`-name`):**
    * `$ find /home/user -name "report.txt"`
    * (user ၏ home ထဲတွင် report.txt အမည်ရှိသော file ကို ရှာမည်)
* **စာလုံး အကြီးအသေး မရွေး ရှာဖွေခြင်း (`-iname`):**
    * `$ find . -iname "*.JPG"`
    * (လက်ရှိ directory ထဲတွင် .jpg သို့မဟုတ် .JPG အဆုံးသတ်သမျှကို ရှာမည်)
* **File အမျိုးအစားဖြင့် ရှာဖွေခြင်း (`-type`):**
    * `$ find /etc -type d` (Directory များကိုသာ ရှာရန်)
    * `$ find /etc -type f` (Regular file များကိုသာ ရှာရန်)
* **File အရွယ်အစားဖြင့် ရှာဖွေခြင်း (`-size`):**
    * `$ find /var/log -size +10M`
    * (10 Megabytes ထက်ကြီးသော file များကို ရှာမည်)

#### Output Redirection with `find`
`find` command ကို အသုံးပြုသည့်အခါ ခွင့်ပြုချက်မရှိသော directory များထဲသို့ ဝင်ရောက်ရှာဖွေပါက "Permission denied" error များ တက်လာနိုင်သည်။ ထို error များကို ဖယ်ထုတ်ပြီး ရလဒ်ကောင်းများကိုသာ ကြည့်လိုပါက **Standard Error Redirection (`2>`)** ကို အသုံးပြုနိုင်သည် -

* **ဥပမာ:** `$ find /etc -name "passwd" 2> /dev/null`
* ဤနေရာတွင် `2> /dev/null` သည် တက်လာသမျှ error များကို "black hole" သို့ ပို့ပေးလိုက်ခြင်း ဖြစ်သဖြင့် error message များ မမြင်ရတော့ဘဲ ရှာတွေ့သည့် result ကိုသာ ပြပေးမည်ဖြစ်သည်။

### အကျဉ်းချုပ် (Summary)
ဤသင်ခန်းစာတွင် သင်ယူခဲ့ရသည်များမှာ-
* `--help` ကို သုံး၍ command တစ်ခု၏ အသုံးပြုပုံ အကျဉ်းကို ကြည့်ရှုနည်း။
* `man` နှင့် `info` တို့ကို သုံး၍ အသေးစိတ် လက်စွဲစာအုပ်များကို ဖတ်ရှုနည်း။
* `/usr/share/doc/` အောက်ရှိ software documentation များအကြောင်း။
* `which`, `whereis` နှင့် `locate` တို့ကို သုံး၍ system အတွင်း file များကို ရှာဖွေနည်း။
* `find` command ကို အသုံးပြု၍ criteria ပေါင်းစုံဖြင့် file များ ရှာဖွေနည်း။
