* **အလေးချိန် (Weight):** 2

### နိဒါန်း (Introduction)
Linux နှင့် အခြား Open Source software များ၏ အောင်မြင်မှုသည် ၎င်းတို့၏ နည်းပညာကြောင့်သာမက ၎င်းတို့ကို ထိန်းချုပ်ထားသော Licensing (လိုင်စင်) မူဝါဒများကြောင့်လည်း ဖြစ်သည်။ ဤသင်ခန်းစာတွင် Free and Open Source Software (FOSS) ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက်များ၊ လိုင်စင်အမျိုးအစားများနှင့် Open Source ကို အခြေခံသည့် စီးပွားရေးပုံစံများကို လေ့လာရမည်ဖြစ်သည်။

### Free and Open Source Software (FOSS) ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက်
"Free Software" ဆိုရာတွင် "အခမဲ့" ဟု တစ်ထိုက်တည်း မဆိုလိုဘဲ "လွတ်လပ်ခွင့်" (Freedom) ကို ဆိုလိုခြင်းဖြစ်သည်။ **Free Software Foundation (FSF)** ၏ တည်ထောင်သူ Richard Stallman က အောက်ပါ လွတ်လပ်ခွင့် ၄ ချက်ကို အဓိပ္ပာယ်ဖွင့်ဆိုခဲ့သည်-

1.  **Freedom 0:** ပရိုဂရမ်ကို မည်သည့် ရည်ရွယ်ချက်အတွက်မဆို အသုံးပြုခွင့်။
2.  **Freedom 1:** ပရိုဂရမ် မည်သို့ အလုပ်လုပ်သည်ကို လေ့လာခွင့်နှင့် မိမိလိုအပ်သလို ပြုပြင်ခွင့် (Source code ကို ကြည့်ရှုခွင့် လိုအပ်သည်)။
3.  **Freedom 2:** မိမိ၏ အိမ်နီးချင်း သို့မဟုတ် အခြားသူများကို ကူညီနိုင်ရန် မိတ္တူများ ပြန်လည် ဝေမျှခွင့်။
4.  **Freedom 3:** မိမိ ပြုပြင်ထားသော version များကို အများပြည်သူသို့ ပြန်လည် ဖြန့်ဝေခွင့်။

---

### Licenses (လိုင်စင်များ)
Open Source လောကတွင် အဓိကအားဖြင့် လိုင်စင် အမျိုးအစား နှစ်မျိုးရှိသည်-

#### 1. Copyleft Licenses
ဤလိုင်စင်အောက်တွင် ရှိသော software ကို ပြုပြင်မွမ်းမံပြီး ပြန်လည်ဖြန့်ဝေပါက ထိုအသစ်ထွက်လာသော software သည်လည်း မူရင်းလိုင်စင်အတိုင်း Open Source ဖြစ်ရမည်ဟု သတ်မှတ်ထားသည်။
* **GNU General Public License (GPL):** အသုံးအများဆုံး Copyleft လိုင်စင်ဖြစ်သည်။ Linux kernel ကို GPLv2 အောက်တွင် လိုင်စင်ပြုထားသည်။
* **Lesser GPL (LGPL):** အခြား software library များဖြင့် ချိတ်ဆက်အသုံးပြုနိုင်ရန် အနည်းငယ် လျှော့ပေါ့ပေးထားသော လိုင်စင်ဖြစ်သည်။

#### 2. Permissive Licenses
ဤလိုင်စင်များသည် အလွန်လွတ်လပ်ပြီး software ကို ပြုပြင်မွမ်းမံပြီးနောက် source code ပြန်ပေးရန် မလိုဘဲ proprietary (မူပိုင်ခွင့်ရှိ software) အဖြစ် ပြောင်းလဲရောင်းချခွင့်ကိုပါ ခွင့်ပြုထားသည်။
* **BSD License:** အလွန် ရိုးရှင်းသော လိုင်စင်ဖြစ်သည်။
* **MIT License:** အလွန် လွတ်လပ်ပြီး developer များကြား ရေပန်းစားသည်။
* **Apache License:** Software နှင့်အတူ patent (မူပိုင်ခွင့်) အခွင့်အရေးများကိုပါ ထည့်သွင်းစဉ်းစားထားသည်။

---

### Creative Commons (CC)
Software များအတွက် GPL သို့မဟုတ် BSD လိုင်စင်များကို အသုံးပြုသော်လည်း စာသားများ၊ ဓာတ်ပုံများ၊ ဗီဒီယိုများနှင့် အနုပညာလက်ရာများအတွက်မူ **Creative Commons (CC)** လိုင်စင်များကို အသုံးပြုကြသည်။ Creative Commons သည် ဖန်တီးသူများအနေဖြင့် ၎င်းတို့၏ လက်ရာများကို မည်ကဲ့သို့ ပြန်လည်အသုံးပြုနိုင်သည်ကို သတ်မှတ်ရန် ရွေးချယ်စရာ ၄ မျိုး ပေးထားသည်-

* **Attribution (BY):** ဖန်တီးသူ မည်သူဖြစ်ကြောင်း အသိအမှတ်ပြု ဖော်ပြရမည်။
* **ShareAlike (SA):** မူရင်းလက်ရာကို ပြုပြင်မွမ်းမံပြီး ပြန်လည်ဖြန့်ဝေပါက မူရင်းလိုင်စင်အတိုင်းသာ ပြန်လည်ဖြန့်ဝေရမည်။
* **Non-Commercial (NC):** စီးပွားရေး ရည်ရွယ်ချက်အတွက် အသုံးမပြုရ။
* **No-Derivatives (ND):** မူရင်းအတိုင်းသာ အသုံးပြုရမည်ဖြစ်ပြီး ပြုပြင်မွမ်းမံခွင့် မပြုပါ။

---

### Free Software နှင့် Open Source အယူအဆ ကွာခြားချက်
Free Software နှင့် Open Source နှစ်ခုလုံးသည် source code ကို ကြည့်ရှုခွင့်ပေးသော်လည်း ၎င်းတို့၏ ဦးတည်ချက်မှာ ကွဲပြားသည်-

* **Free Software:** ၎င်းသည် ကိုယ်ကျင့်တရားနှင့် လူမှုရေးဆိုင်ရာ (Moral/Social) ကိစ္စရပ်အဖြစ် ရှုမြင်သည်။ အသုံးပြုသူ၏ လွတ်လပ်ခွင့်ကို အဓိကထားသည်။
* **Open Source:** ၎င်းသည် software ဖွံ့ဖြိုးတိုးတက်ရေး နည်းလမ်း (Development Methodology) အဖြစ် ရှုမြင်သည်။ Source code ကို ဖွင့်ပေးထားခြင်းဖြင့် software သည် ပိုမိုကောင်းမွန်ပြီး အမှား (bug) ကင်းစင်မည်ဟု ယူဆသည်။

ဤနှစ်ခုလုံးကို ပေါင်းစပ်၍ **FOSS (Free and Open Source Software)** သို့မဟုတ် **FLOSS (Free/Libre and Open Source Software)** ဟု ခေါ်ဝေါ်ကြသည်။

---

### Open Source Philosophy (အယူအဆများ)
Open Source ၏ အဓိက အယူအဆများမှာ-
* **Transparency:** အရာအားလုံးကို ပွင့်လင်းမြင်သာစွာ လုပ်ဆောင်သည်။
* **Collaboration:** ကမ္ဘာအနှံ့မှ developer များ စုပေါင်းအလုပ်လုပ်ကြသည်။
* **Rapid Prototyping:** Software version အသစ်များကို လျင်မြန်စွာ ထုတ်ဝေပြီး စမ်းသပ်သည်။
* **Meritocracy:** အလုပ်အပေါ် မူတည်၍ ဆုံးဖြတ်ချက်ချသူများကို ရွေးချယ်သည်။

---

### Open Source Business Models (စီးပွားရေး ပုံစံများ)
Open Source software များသည် အခမဲ့ ရရှိနိုင်သော်လည်း ကုမ္ပဏီများသည် ၎င်းတို့ကို အခြေခံ၍ ငွေကြေး ရှာဖွေနိုင်သည့် ပုံစံများစွာ ရှိသည်-

* **Support and Service:** Software ကို အခမဲ့ ပေးသော်လည်း ၎င်းကို တပ်ဆင်ခြင်း၊ ထိန်းသိမ်းခြင်းနှင့် နည်းပညာ အကူအညီ (Technical support) ပေးခြင်းအတွက် ဝန်ဆောင်ခ ယူခြင်း (ဥပမာ - Red Hat)။
* **Professional Services:** ကုမ္ပဏီတစ်ခု၏ လိုအပ်ချက်နှင့် ကိုက်ညီအောင် software ကို အထူးပြုပြင် (Customization) ပေးခြင်း။
* **Dual Licensing:** Software ကို Open Source အဖြစ်လည်း ပေးထားပြီး၊ စီးပွားရေးလုပ်ငန်းသုံး (Commercial) အတွက် အခြား လိုင်စင်တစ်ခုဖြင့် သီးခြား ရောင်းချခြင်း (ဥပမာ - MySQL)။
* **SaaS (Software as a Service):** Software ကို cloud ပေါ်တွင် တင်ထားပြီး အသုံးပြုသူများထံမှ လစဉ်ကြေး သို့မဟုတ် အသုံးပြုသလောက် ပေးချေစနစ်ဖြင့် ငွေယူခြင်း။

---

### အကျဉ်းချုပ် (Summary)
ဤသင်ခန်းစာတွင် သင်ယူခဲ့ရသည်များမှာ-
* Free Software ၏ အခြေခံ လွတ်လပ်ခွင့် ၄ ချက်။
* Copyleft နှင့် Permissive လိုင်စင်များ၏ ကွာခြားချက်။
* GPL, BSD နှင့် Apache လိုင်စင်များအကြောင်း။
* Creative Commons (CC) လိုင်စင်များအကြောင်း။
* Open Source software ကို အသုံးပြု၍ စီးပွားရေးလုပ်ဆောင်နိုင်သည့် နည်းလမ်းများ။
