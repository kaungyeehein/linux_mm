### နိဒါန်း (Introduction)
ဤသင်ခန်းစာတွင် Free နှင့် Open Source Software လိုင်စင်များ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက်များ၊ ၎င်းတို့၏ ကွာခြားချက်များနှင့် Open Source လောကတွင် အသုံးပြုနေသော စီးပွားရေးပုံစံများကို လေ့လာရမည် ဖြစ်သည်။

### Free နှင့် Open Source Software လိုင်စင်များ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက် (Definition of Free and Open Source Software Licenses)

Software တစ်ခုကို "Free" သို့မဟုတ် "Open Source" ဟု သတ်မှတ်ရန်အတွက် အခြေခံလွတ်လပ်ခွင့်များနှင့် စည်းမျဉ်းများ ရှိရပါမည်။ ဤနယ်ပယ်တွင် အဓိကကျသော အဖွဲ့အစည်းကြီးနှစ်ခုမှာ Free Software Foundation (FSF) နှင့် Open Source Initiative (OSI) တို့ ဖြစ်ကြသည်။

#### Free Software (လွတ်လပ်သော ဆော့ဖ်ဝဲလ်)
Free Software Foundation (FSF) ၏ အဆိုအရ ဆော့ဖ်ဝဲလ်အသုံးပြုသူများအတွက် အခြေခံလွတ်လပ်ခွင့် (၄) ချက် ရှိရမည် ဖြစ်သည်-
- **လွတ်လပ်ခွင့် ၀ (Freedom 0):** မည်သည့် ရည်ရွယ်ချက်အတွက်မဆို ပရိုဂရမ်ကို အသုံးပြုခွင့်။
- **လွတ်လပ်ခွင့် ၁ (Freedom 1):** ပရိုဂရမ်၏ အလုပ်လုပ်ပုံကို လေ့လာခွင့်နှင့် မိမိလိုအပ်သလို ပြုပြင်ပြောင်းလဲခွင့် (Source code ကို ကြည့်ရှုခွင့်ရှိရမည်)။
- **လွတ်လပ်ခွင့် ၂ (Freedom 2):** မိမိ၏ အိမ်နီးချင်းများ သို့မဟုတ် အခြားသူများကို ကူညီနိုင်ရန် မိတ္တူများ ပြန်လည်ဖြန့်ဝေခွင့်။
- **လွတ်လပ်ခွင့် ၃ (Freedom 3):** မိမိပြုပြင်ထားသော ဗားရှင်းများကို အများသူငါထံသို့ ပြန်လည်ဖြန့်ဝေခွင့်။

#### Open Source Software (အရင်းအမြစ်ဖွင့် ဆော့ဖ်ဝဲလ်)
Open Source Initiative (OSI) သည် ဆော့ဖ်ဝဲလ်ကို အများသူငါ ပူးပေါင်းပါဝင် တည်ဆောက်နိုင်ရန်နှင့် စီးပွားရေးအရ အသုံးပြုနိုင်ရန် ပိုမိုအာရုံစိုက်သည်။ Open Source လိုင်စင်တစ်ခုတွင် ပါဝင်ရမည့် အချက်များမှာ-
- အခမဲ့ ပြန်လည်ဖြန့်ဝေခွင့် (Free Redistribution)။
- Source Code ကို အလွယ်တကူ ရရှိနိုင်မှု။
- မူရင်းဆော့ဖ်ဝဲလ်ကို အခြေခံ၍ အသစ်တစ်ခု ထပ်မံဖန်တီးခွင့် (Derived Works)။
- မည်သည့်ပုဂ္ဂိုလ် သို့မဟုတ် အုပ်စုကိုမျှ ခွဲခြားဆက်ဆံမှု မရှိစေခြင်း။

### လိုင်စင်အမျိုးအစားများ (Types of Licenses)

Open Source လောကတွင် လိုင်စင်များကို အဓိကအားဖြင့် အမျိုးအစား (၂) မျိုး ခွဲခြားနိုင်သည်-

1.  **Copyleft Licenses (GPL ကဲ့သို့သော):**
    - ဤလိုင်စင်အောက်ရှိ ဆော့ဖ်ဝဲလ်ကို ပြုပြင်ပြောင်းလဲ၍ ပြန်လည်ဖြန့်ဝေပါက ထိုအသစ်ထွက်လာသော ဆော့ဖ်ဝဲလ်သည်လည်း မူရင်းလိုင်စင် (GPL) အတိုင်းသာ ဖြစ်ရမည်ဟု သတ်မှတ်ထားသည်။
    - ဥပမာ - GNU General Public License (GPL)။

2.  **Permissive Licenses (ခွင့်ပြုချက် ကျယ်ပြန့်သော လိုင်စင်များ):**
    - အသုံးပြုသူများအား အလွန်ကျယ်ပြန့်သော ခွင့်ပြုချက်များ ပေးထားသည်။ ပြုပြင်ထားသော ဆော့ဖ်ဝဲလ်ကို Open Source အဖြစ် ဆက်လက်ထားရှိရန် မလိုအပ်ဘဲ Proprietary (ပိုင်ရှင်ရှိ ဆော့ဖ်ဝဲလ်) အဖြစ်ပင် ပြောင်းလဲနိုင်သည်။
    - ဥပမာ - BSD License, MIT License, Apache License။

---

### Creative Commons (CC)
Creative Commons သည် ဆော့ဖ်ဝဲလ်များအတွက် မဟုတ်ဘဲ စာသားများ၊ ဓာတ်ပုံများနှင့် ဗီဒီယိုများကဲ့သို့သော ဖန်တီးမှုလက်ရာများ (Creative Works) အတွက် အသုံးပြုသည့် လိုင်စင်ဖြစ်သည်။ ၎င်းတွင် ပုံစံအမျိုးမျိုး ရှိသည်-
- **BY (Attribution):** မူရင်းဖန်တီးသူကို အသိအမှတ်ပြု ဖော်ပြရမည်။
- **SA (Share Alike):** ပြန်လည်ဖြန့်ဝေပါက မူရင်းလိုင်စင်အတိုင်းသာ အသုံးပြုရမည်။
- **NC (Non-Commercial):** စီးပွားရေးအတွက် အသုံးမပြုရ။
- **ND (No Derivatives):** မူရင်းလက်ရာကို ပြုပြင်ပြောင်းလဲခြင်း မပြုရ။
