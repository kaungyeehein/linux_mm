### နိဒါန်း (Introduction)

Open Source ဟူသော ဝေါဟာရသည် software တစ်ခု၏ မူရင်း code (source code) ကို မည်သူမဆို လေ့လာနိုင်ရန်၊ ပြုပြင်နိုင်ရန်နှင့် ပြန်လည်ဖြန့်ဝေနိုင်ရန် ခွင့်ပြုထားခြင်းကို ဆိုလိုသည်။ ၎င်းသည် software ဖန်တီးပုံ နည်းလမ်းတစ်ခုသာမကဘဲ ပူးပေါင်းဆောင်ရွက်ခြင်းနှင့် ဗဟုသုတ မျှဝေခြင်းဆိုင်ရာ အတွေးအခေါ်တစ်ခုလည်း ဖြစ်သည်။ Open Source လောကတွင် အဓိကကျသော အဖွဲ့အစည်းကြီး နှစ်ခုရှိသည်- Free Software Foundation (FSF) နှင့် Open Source Initiative (OSI) တို့ဖြစ်သည်။

### Free Software Foundation (FSF) နှင့် လွတ်လပ်ခွင့် ၄ မျိုး

FSF ကို ၁၉၈၅ ခုနှစ်တွင် Richard Stallman မှ တည်ထောင်ခဲ့သည်။ FSF ၏ အဓိက ရည်ရွယ်ချက်မှာ အသုံးပြုသူများ၏ လွတ်လပ်ခွင့်ကို ကာကွယ်ရန်ဖြစ်သည်။ ၎င်းတို့သည် software တစ်ခုကို "Free Software" ဟု သတ်မှတ်နိုင်ရန်အတွက် အောက်ပါ လွတ်လပ်ခွင့် ၄ မျိုး ရှိရမည်ဟု သတ်မှတ်ထားသည်-

၁။ မည်သည့် ရည်ရွယ်ချက်အတွက်မဆို program ကို run နိုင်ခွင့် (Freedom 0)။  
၂။ Program အလုပ်လုပ်ပုံကို လေ့လာနိုင်ခွင့်နှင့် မိမိလိုအပ်သလို ပြုပြင်နိုင်ခွင့် (Freedom 1)။  
၃။ အခြားသူများကို ကူညီနိုင်ရန်အတွက် software မိတ္တူများကို ပြန်လည်ဖြန့်ဝေနိုင်ခွင့် (Freedom 2)။  
၄။ မိမိပြုပြင်ထားသော ဗားရှင်းကို အများပြည်သူထံ ဖြန့်ဝေနိုင်ခွင့် (Freedom 3)။  

### Open Source Initiative (OSI)

OSI ကို ၁၉၉၈ ခုနှစ်တွင် တည်ထောင်ခဲ့ပြီး "Open Source" ဟူသော ဝေါဟာရကို စီးပွားရေးလောကတွင် ပိုမိုလက်ခံလာစေရန် ရည်ရွယ်သည်။ OSI သည် software ၏ အရည်အသွေးနှင့် ပူးပေါင်းဆောင်ရွက်မှုကို ပိုမိုအသားပေးသည်။ ၎င်းတို့သည် "Open Source Definition" ဟုခေါ်သော စည်းမျဉ်း ၁၀ ချက်ကို သတ်မှတ်ထားပြီး ထိုစည်းမျဉ်းများနှင့် ကိုက်ညီသော လိုင်စင်များကိုသာ Open Source လိုင်စင်အဖြစ် အသိအမှတ်ပြုသည်။

### Definition of Free and Open Source Software (Free နှင့် Open Source Software ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက်)

"Free Software" နှင့် "Open Source Software" ဟူသော ဝေါဟာရနှစ်ခုလုံးသည် software ၏ source code ကို လူတိုင်းရယူကြည့်ရှုနိုင်ပြီး လွတ်လပ်စွာ ပြုပြင်အသုံးပြုနိုင်ခြင်းကို ဆိုလိုသည်။ သို့သော် ၎င်းတို့၏ အဓိက ကွာခြားချက်မှာ ဦးတည်ချက် (philosophy) ပေါ်တွင် မူတည်သည်။

* **Free Software:** အသုံးပြုသူ၏ ကျင့်ဝတ်ဆိုင်ရာ လွတ်လပ်ခွင့် (Ethical Freedom) ကို အဓိကထားသည်။ ၎င်းသည် software ကို အသုံးပြုသူများအတွက် "လွတ်လပ်ခွင့် ၄ မျိုး" ကို အာမခံချက်ပေးနိုင်ရန် ကြိုးပမ်းသည်။
* **Open Source Software:** လက်တွေ့ကျကျ ပူးပေါင်းဆောင်ရွက်ခြင်းနှင့် software ၏ အရည်အသွေး (Development Methodology) ကို ပိုမိုအလေးပေးသည်။ ကုမ္ပဏီများနှင့် developer များ စုပေါင်း၍ ပိုမိုကောင်းမွန်သော software များ ထုတ်လုပ်နိုင်ရန် ရည်ရွယ်သည်။

ဤအယူအဆ နှစ်ခုလုံးကို ပေါင်းစပ်၍ **FOSS** (Free and Open Source Software) သို့မဟုတ် **FLOSS** (Free/Libre and Open Source Software) ဟုလည်း ခေါ်ဝေါ်ကြသည်။

### Licensing (လိုင်စင်သတ်မှတ်ခြင်း)

Open Source software တိုင်းတွင် ၎င်းကို မည်သို့အသုံးပြုရမည်၊ မည်သို့ပြန်လည်ဖြန့်ဝေရမည်ဟူသော စည်းကမ်းချက်များ (Licensing) ပါရှိသည်။ အဓိကအားဖြင့် အုပ်စုကြီးနှစ်ခု ခွဲခြားနိုင်သည်-

#### ၁။ Copyleft Licenses
ဤလိုင်စင် အမျိုးအစားသည် "လွတ်လပ်ခွင့်ကို လွတ်လပ်ခွင့်ဖြင့် ဆက်ခံခြင်း" ဖြစ်သည်။ အကယ်၍ သင်သည် Copyleft လိုင်စင်ရှိသော software ကို ယူသုံးပြီး ပြုပြင်ပြောင်းလဲမှုများ ပြုလုပ်ခဲ့လျှင်၊ သင်၏ ပြုပြင်ထားသော version ကိုလည်း မူရင်းလိုင်စင်အတိုင်း (Open Source အဖြစ်သာ) ပြန်လည်ဖြန့်ဝေရမည်ဖြစ်သည်။
* **GPL (GNU General Public License):** အသုံးအများဆုံး Copyleft လိုင်စင်ဖြစ်သည်။ Linux Kernel သည် ဤလိုင်စင်ကို အသုံးပြုထားသည်။
* **LGPL (Lesser GPL):** ၎င်းသည် software library များအတွက် အဓိကထားသည်။ Proprietary software များမှလည်း ဤ library များကို ချိတ်ဆက်အသုံးပြုခွင့် ပေးထားသည်။

#### ၂။ Permissive Licenses
ဤလိုင်စင်များသည် အလွန်လွတ်လပ်သည်။ Software ကို ယူသုံးသူအနေဖြင့် ၎င်းကို ပြုပြင်ပြီးနောက် open source အဖြစ် ပြန်လည်ဖြန့်ဝေရန် မလိုသလို၊ မူပိုင်ခွင့် (proprietary) software အဖြစ် ပြောင်းလဲ၍ ရောင်းချခြင်းကိုလည်း ခွင့်ပြုထားသည်။
* **BSD (Berkeley Software Distribution):** အလွန်ရိုးရှင်းသော လိုင်စင်ဖြစ်ပြီး၊ မူရင်းဖန်တီးသူ၏ အမည်ကို ဖော်ပြရုံဖြင့် စိတ်ကြိုက်အသုံးပြုနိုင်သည်။
* **MIT License:** BSD နှင့် ဆင်တူပြီး ခေတ်မီ software project အတော်များများတွင် အသုံးပြုကြသည်။

### Creative Commons (CC)
Software code များအတွက်မဟုတ်ဘဲ စာအုပ်များ၊ ဓာတ်ပုံများ၊ ဗီဒီယိုများနှင့် အနုပညာလက်ရာများအတွက် အသုံးပြုသော လိုင်စင်စနစ်ဖြစ်သည်။ ဥပမာအားဖြင့် ယခုသင်လေ့လာနေသော LPI သင်ခန်းစာများကို Creative Commons လိုင်စင်ဖြင့် ဖြန့်ချိထားခြင်း ဖြစ်သည်။
