### နိဒါန်း (Introduction)
Linux သည် လူကြိုက်အများဆုံး operating system များထဲမှ တစ်ခုဖြစ်သည်။ ၎င်းကို ၁၉၉၁ ခုနှစ်တွင် Linus Torvalds မှ စတင်ဖန်တီးခဲ့ခြင်း ဖြစ်သည်။ ဤ operating system ကို ၁၉၇၀ ပြည့်လွန်နှစ်များအတွင်း AT&T ဓာတ်ခွဲခန်းမှ ထုတ်လုပ်ခဲ့သော Unix အမည်ရှိ အခြား operating system တစ်ခုကို အခြေခံ၍ စိတ်ကူးရယူ တည်ဆောက်ခဲ့ခြင်း ဖြစ်သည်။

Unix သည် ထိုခေတ်အခါက ကွန်ပျူတာအသေးစားများအတွက် ရည်ရွယ်ထုတ်လုပ်ခဲ့ခြင်း ဖြစ်သည်။ ထိုအချိန်က "အသေးစား" ကွန်ပျူတာဆိုသည်မှာ လေအေးပေးစနစ်ပါသော ခန်းမတစ်ခုလုံးစာ နေရာယူရန်မလိုဘဲ တန်ဖိုးအားဖြင့် ဒေါ်လာတစ်သန်းအောက်ရှိသော စက်များကို ဆိုလိုခြင်းဖြစ်သည်။ နောက်ပိုင်းတွင် လူနှစ်ယောက် မနိုင်တန်ရုံမျှရှိသော စက်များကိုပါ ဆိုလိုလာကြသည်။

ထိုစဉ်အခါက x86 platform ကို အခြေခံထားသော ရုံးသုံးကွန်ပျူတာများကဲ့သို့သော စက်များပေါ်တွင် သင့်တင့်သောဈေးနှုန်းဖြင့် အသုံးပြုနိုင်သည့် Unix system များ အလွယ်တကူ မရှိသေးပေ။ ထို့ကြောင့် ထိုအချိန်က ကျောင်းသားတစ်ဦးဖြစ်ခဲ့သူ Linus သည် ဤ platform ပေါ်တွင် အသုံးပြုနိုင်မည့် Unix နှင့် ဆင်တူသော (Unix-like) operating system တစ်ခုကို စတင်အကောင်အထည်ဖော်ခဲ့သည်။

အများအားဖြင့် Linux သည် Unix ၏ အခြေခံသဘောတရားများနှင့် မူဝါဒများကိုသာ အသုံးပြုထားခြင်းဖြစ်ပြီး၊ Linux ကိုယ်တိုင်မှာ သီးခြားစီမံကိန်း (independent project) တစ်ခုဖြစ်သောကြောင့် Unix ၏ code များ ပါဝင်ခြင်းမရှိပါ။ Linux ကို ကုမ္ပဏီတစ်ခုတည်းက ထောက်ပံ့ထားခြင်းမျိုး မဟုတ်ဘဲ နိုင်ငံတကာမှ ပရိုဂရမ်မာများ၏ အသိုက်အဝန်း (international community) က ထောက်ပံ့ပေးထားခြင်း ဖြစ်သည်။ ၎င်းကို အခမဲ့ အလွယ်တကူ ရရှိနိုင်သောကြောင့် မည်သူမဆို ကန့်သတ်ချက်မရှိဘဲ အသုံးပြုနိုင်ပါသည်။

### Distributions
Linux ဖြန့်ချိမှုပုံစံများ

Linux သည် kernel တစ်ခုသာ ဖြစ်သောကြောင့် ၎င်းကို အသုံးပြုသူများအတွက် အဆင်သင့်သုံးနိုင်သော operating system တစ်ခုဖြစ်လာစေရန် အခြား software အစိတ်အပိုင်းများနှင့် ပေါင်းစပ်ပေးရသည်။ Linux kernel ကို library များ၊ utility များ၊ application software များ၊ documentation များနှင့် ပေါင်းစပ်ကာ အလုံးစုံပြည်စုံသော operating system တစ်ခုအဖြစ် ထုပ်ပိုးထားခြင်းကို Linux Distribution (သို့မဟုတ်) Distro ဟု ခေါ်ဆိုသည်။

Linux ဖြန့်ချိမှုပုံစံ (Distribution) အမျိုးမျိုးရှိပြီး တစ်ခုနှင့်တစ်ခု ရည်ရွယ်ချက်ချင်း မတူညီကြပါ။ အချို့မှာ desktop ကွန်ပျူတာများအတွက် ဖြစ်ပြီး၊ အချို့မှာ server များ သို့မဟုတ် အထူးပြုလုပ်ဆောင်ချက်များအတွက် ဖြစ်သည်။

#### အဓိက Linux Distribution မိသားစုများ

၁။ **Debian Family:**
* **Debian:** ၎င်းသည် လွတ်လပ်၍ အခမဲ့ဖြစ်သော software များအပေါ်တွင်သာ အခြေခံပြီး တည်ငြိမ်မှု (stability) ကို အဓိကထားသော distribution တစ်ခုဖြစ်သည်။
* **Ubuntu:** Debian အပေါ်တွင် အခြေခံထားပြီး အသုံးပြုရ လွယ်ကူစေရန် ဖန်တီးထားသည်။ ၎င်းတွင် ပုံမှန်ထုတ်ဝေသော version များအပြင် (၅) နှစ်အထိ ရေရှည်ထောက်ပံ့ပေးသော **Long Term Support (LTS)** version များလည်း ရှိသည်။
* **Linux Mint:** Ubuntu အပေါ်တွင် ထပ်ဆင့်အခြေခံထားပြီး desktop အသုံးပြုသူများအတွက် ပိုမိုသက်သောင့်သက်သာရှိစေရန် ပြုပြင်မွမ်းမံထားသည်။

၂။ **Red Hat Family:**
* **Red Hat Enterprise Linux (RHEL):** စီးပွားရေးလုပ်ငန်းကြီးများ (Enterprise) အတွက် ရည်ရွယ်ပြီး အခကြေးငွေဖြင့် ဝန်ဆောင်မှုပေးသော distribution ဖြစ်သည်။
* **CentOS:** RHEL ၏ code ကို အခြေခံ၍ အခမဲ့ ပြန်လည်ထုတ်ဝေထားသော distribution ဖြစ်သည်။
* **Scientific Linux:** သိပ္ပံပညာရှင်များနှင့် သုတေသနလုပ်ငန်းများအတွက် အထူးပြုထုတ်လုပ်ထားသည်။

၃။ **SUSE Family:**
* **SUSE Linux Enterprise (SLE):** စီးပွားရေးလုပ်ငန်းသုံးအတွက် ရည်ရွယ်သော distribution ဖြစ်သည်။
* **openSUSE:** SUSE ၏ အခမဲ့နှင့် community အခြေပြု version ဖြစ်သည်။

#### Distribution တစ်ခုကို ရွေးချယ်ရာတွင် ထည့်သွင်းစဉ်းစားရမည့်အချက်များ

Linux distribution တစ်ခုကို ရွေးချယ်ရာတွင် အောက်ပါအချက်များကို ကြည့်ရှုရမည် -
* **အသုံးပြုမည့် ရည်ရွယ်ချက် (Use case):** Desktop အတွက်လား၊ Server အတွက်လား သို့မဟုတ် Embedded system အတွက်လား။
* **ထောက်ပံ့မှု သက်တမ်း (Support cycle):** LTS ကဲ့သို့သော ရေရှည်ထောက်ပံ့မှု လိုအပ်ပါသလား။
* **Software များ ရရှိနိုင်မှု (Software availability):** မိမိအသုံးပြုလိုသော software များကို အလွယ်တကူ install လုပ်နိုင်ပါသလား။
* **Hardware ထောက်ပံ့မှု (Hardware support):** မိမိစက်ရှိ hardware များနှင့် ကိုက်ညီမှု ရှိပါသလား။

### Embedded Systems
မြှုပ်နှံထားသော စနစ်များ

Embedded System ဆိုသည်မှာ သီးခြားလုပ်ဆောင်ချက်တစ်ခု သို့မဟုတ် အကန့်အသတ်ရှိသော လုပ်ဆောင်ချက်အနည်းငယ်ကိုသာ ဆောင်ရွက်ရန်အတွက် စက်ပစ္စည်းတစ်ခုအတွင်း ထည့်သွင်းတည်ဆောက်ထားသော ကွန်ပျူတာစနစ်တစ်ခုဖြစ်သည်။ ဥပမာအားဖြင့် အိမ်သုံးလျှပ်စစ်ပစ္စည်းများ၊ ကားများ သို့မဟုတ် စက်မှုလုပ်ငန်းသုံး စက်ကိရိယာများတွင် တွေ့ရတတ်သည်။

Linux သည် Embedded System များအတွက် အလွန်သင့်လျော်သော Operating System တစ်ခုဖြစ်သည်။ အဘယ်ကြောင့်ဆိုသော် Linux ကို မိမိအသုံးပြုလိုသော စက်ပစ္စည်း၏ လိုအပ်ချက်နှင့်အညီ လိုအပ်သလို ပြုပြင်ပြောင်းလဲနိုင်ခြင်း (Customizable) နှင့် အရင်းအမြစ် အနည်းငယ်သာရှိသော စက်ပစ္စည်းများတွင်လည်း ကောင်းမွန်စွာ လည်ပတ်နိုင်ခြင်းတို့ကြောင့် ဖြစ်သည်။

#### အသုံးများသော Embedded Linux နမူနာများ

* **Android:** Linux kernel အပေါ်တွင် အခြေခံထားသော လူကြိုက်အများဆုံး mobile operating system ဖြစ်သည်။ ၎င်းကို စမတ်ဖုန်းများ၊ တက်ဘလက်များနှင့် အခြားသော smart device များတွင် အသုံးပြုသည်။
* **Raspberry Pi:** ၎င်းသည် သေးငယ်ပြီး ဈေးနှုန်းချိုသာသော ကွန်ပျူတာအမျိုးအစားဖြစ်ပြီး Linux (ဥပမာ- Raspbian သို့မဟုတ် Raspberry Pi OS) ကို အသုံးပြုကာ ပညာရေးဆိုင်ရာနှင့် ဝါသနာရှင်များ၏ စီမံကိန်းများတွင် အသုံးများသည်။
* **Smart Appliances:** TV များ၊ ရေခဲသေတ္တာများနှင့် အိမ်သုံး router များစွာတွင်လည်း Linux ကို Embedded System အဖြစ် အသုံးပြုထားကြသည်။

### Linux and the Cloud
Linux နှင့် ကလောက် (Cloud)

Cloud Computing ဆိုသည်မှာ အင်တာနက်မှတစ်ဆင့် server များ၊ storage များ၊ database များနှင့် software ဝန်ဆောင်မှုများကို လိုအပ်သလို ရယူအသုံးပြုခြင်း ဖြစ်သည်။ ယနေ့ခေတ် cloud အခြေခံအဆောက်အအုံ (infrastructure) အများစုသည် Linux ပေါ်တွင် အဓိက အခြေခံထားသည်။

#### Cloud တွင် Linux ကို အသုံးပြုရခြင်း အကြောင်းရင်းများ

၁။ **စွမ်းဆောင်ရည်နှင့် တည်ငြိမ်မှု:** Linux သည် server များပေါ်တွင် အလွန်တည်ငြိမ်ပြီး စွမ်းဆောင်ရည်မြင့်မားစွာ လည်ပတ်နိုင်သောကြောင့် cloud provider များ (ဥပမာ- Amazon Web Services, Google Cloud, Microsoft Azure) က အဓိကအသုံးပြုကြသည်။

၂။ **Scalability (ချဲ့ထွင်နိုင်စွမ်း):** Cloud ပတ်ဝန်းကျင်တွင် လိုအပ်ချက်အရ server အသစ်များကို အချိန်တိုအတွင်း ထောင်ပေါင်းများစွာ တိုးချဲ့အသုံးပြုရန် Linux က လွယ်ကူစွာ ပံ့ပိုးပေးနိုင်သည်။

၃။ **Cost Efficiency (ကုန်ကျစရိတ် သက်သာခြင်း):** Linux သည် open source ဖြစ်သောကြောင့် license ကြေးများ ပေးဆောင်ရန် မလိုဘဲ အသုံးပြုနိုင်ခြင်းက cloud ဝန်ဆောင်မှုစရိတ်များကို လျှော့ချပေးနိုင်သည်။

၄။ **Containerization:** ခေတ်မီသော cloud နည်းပညာများဖြစ်သည့် Docker နှင့် Kubernetes တို့သည် Linux kernel ၏ အင်္ဂါရပ်များကို အသုံးပြုထားခြင်းဖြစ်သောကြောင့် cloud-native application များအတွက် Linux သည် မရှိမဖြစ် လိုအပ်ပါသည်။

### Answers to Guided Exercises
လမ်းညွှန်လေ့ကျင့်ခန်းများအတွက် အဖြေများ

၁။ Linux distribution များကို ၎င်းတို့၏ မိသားစု (Family) အလိုက် တွဲပေးခြင်း-
   * **Debian Family:** Debian, Ubuntu
   * **Red Hat Family:** CentOS, RHEL
   * **SUSE Family:** openSUSE

၂။ Android operating system သည် **Linux kernel** ကို အခြေခံ၍ တည်ဆောက်ထားခြင်းဖြစ်သည်။

၃။ Raspberry Pi ကွန်ပျူတာများတွင် အသုံးအများဆုံး operating system မှာ **Raspbian (Raspberry Pi OS)** ဖြစ်သည်။

### Answers to Explorational Exercises
ရှာဖွေလေ့လာရန် လေ့ကျင့်ခန်းများအတွက် အဖြေများ

၁။ သင်၏ ကွန်ပျူတာ သို့မဟုတ် မိုဘိုင်းလ်ဖုန်းတွင် အသုံးပြုနေသော operating system အကြောင်း-
   * အကယ်၍ သင်သည် Android ဖုန်းကို အသုံးပြုနေပါက ၎င်းသည် Linux kernel ကို အခြေခံထားပါသည်။
   * အကယ်၍ သင်သည် Apple (iPhone သို့မဟုတ် Mac) ကို အသုံးပြုနေပါက ၎င်းသည် Unix-like ဖြစ်သော်လည်း Linux မဟုတ်ပါ။
   * Windows သည် Linux kernel ကို အခြေခံထားခြင်းမရှိသော်လည်း WSL (Windows Subsystem for Linux) မှတစ်ဆင့် Linux ကို အသုံးပြုနိုင်သည်။

၂။ Linux Distribution Timeline အကြောင်း-
   * Linux သည် Unix ၏ source code ကို အသုံးမပြုဘဲ သီးခြားလွတ်လပ်စွာ ဖန်တီးခဲ့သော်လည်း Unix ၏ အခြေခံဒီဇိုင်းနှင့် မူဝါဒများကို လိုက်နာထားသည့် "Unix-like" စနစ်ဖြစ်သည်။
   * Timeline ကို ကြည့်ရှုခြင်းဖြင့် Debian, Slackware နှင့် Red Hat ကဲ့သို့သော အစောပိုင်း distribution များမှတစ်ဆင့် ယနေ့ခေတ် ခေတ်စားနေသော Ubuntu နှင့် အခြား distro များ မည်သို့ဆင်းသက်လာသည်ကို မြင်တွေ့နိုင်သည်။

### Summary (အနှစ်ချုပ်)

ဤသင်ခန်းစာတွင် သင်ယူခဲ့ရသည်မှာ -
* **Linux ၏ မူလအစ:** Linux ကို ၁၉၉၁ ခုနှစ်တွင် Linus Torvalds မှ Unix ကို အခြေခံ၍ စတင်ဖန်တီးခဲ့သည်။
* **Distributions:** Linux kernel ကို အခြေခံ software များနှင့် စုစည်းထားသော distro အမျိုးမျိုး (ဥပမာ - Ubuntu, Fedora, Debian) ရှိသည်။
* **Embedded Systems:** Linux ကို ဖုန်းများ (Android)၊ အိမ်သုံးပစ္စည်းများနှင့် Raspberry Pi ကဲ့သို့သော စက်ပစ္စည်းအသေးစားများတွင် အသုံးပြုသည်။
* **Cloud Computing:** အင်တာနက်ပေါ်ရှိ cloud ဝန်ဆောင်မှုအများစုသည် ၎င်းတို့၏ စနစ်များကို လည်ပတ်ရန် Linux ကို အဓိက အသုံးပြုကြသည်။
* **Community:** Linux သည် ကုမ္ပဏီတစ်ခုတည်းပိုင်မဟုတ်ဘဲ ကမ္ဘာတစ်ဝှမ်းရှိ developer များ စုပေါင်းပံ့ပိုးထားသော Open Source project တစ်ခုဖြစ်သည်။
