### နိဒါန်း (Introduction)

Linux သည် လူကြိုက်အများဆုံး operating system များထဲမှ တစ်ခုဖြစ်သည်။ ၎င်းကို ၁၉၉၁ ခုနှစ်တွင် Linus Torvalds မှ စတင်ဖော်ထုတ်ခဲ့ခြင်း ဖြစ်သည်။ ဤ operating system ကို ၁၉၇၀ ပြည့်လွန်နှစ်များအတွင်း AT&T Laboratories မှ ထုတ်လုပ်ခဲ့သော Unix အမည်ရှိ အခြား operating system တစ်ခုကို အခြေခံ၍ စိတ်ကူးရရှိခဲ့ခြင်း ဖြစ်သည်။ Unix ကို ကွန်ပျူတာအသေးစားများအတွက် ရည်ရွယ်ခဲ့သည်။ ထိုခေတ်အခါက "အသေးစား" ကွန်ပျူတာဆိုသည်မှာ လေအေးပေးစက်တပ်ဆင်ထားသော ခန်းမတစ်ခုလုံးစာ နေရာယူရန်မလိုဘဲ တန်ဖိုးအားဖြင့် ဒေါ်လာတစ်သန်းအောက်ရှိသော စက်များကို ဆိုလိုခြင်းဖြစ်သည်။ နောက်ပိုင်းတွင် လူနှစ်ယောက် မနိုင်တန်စက်များကို အသေးစားဟု သတ်မှတ်လာကြသည်။

ထိုအချိန်တွင် x86 platform ကို အခြေခံလေ့ရှိသော ရုံးသုံးကွန်ပျူတာများပေါ်တွင် သင့်တင့်သောဈေးနှုန်းဖြင့် အသုံးပြုနိုင်သည့် Unix system မရှိသေးပေ။ ထို့ကြောင့် ထိုစဉ်က ကျောင်းသားတစ်ဦးဖြစ်ခဲ့သော Linus သည် ဤ platform ပေါ်တွင် အလုပ်လုပ်နိုင်မည့် Unix နှင့်ဆင်တူသော operating system တစ်ခုကို စတင်အကောင်အထည်ဖော်ခဲ့သည်။ Linux သည် Unix ၏ မူဝါဒများနှင့် အခြေခံစိတ်ကူးစိတ်သန်းများကို အများဆုံးအသုံးပြုထားသော်လည်း၊ ၎င်းကိုယ်တိုင်သည် သီးခြားလွတ်လပ်သော project တစ်ခုဖြစ်သောကြောင့် Unix code များ ပါဝင်ခြင်းမရှိပါ။ Linux ကို ကုမ္ပဏီတစ်ခုတည်းက ပံ့ပိုးထားခြင်းမျိုးမဟုတ်ဘဲ နိုင်ငံတကာမှ programmer များပါဝင်သော community တစ်ခုမှ ပံ့ပိုးထားခြင်း ဖြစ်သည်။ ၎င်းကို အခမဲ့ရရှိနိုင်သောကြောင့် မည်သူမဆို ကန့်သတ်ချက်မရှိ အသုံးပြုနိုင်သည်။

### Distributions (Linux ဖြန့်ချိမှုပုံစံများ)

Linux distribution ဆိုသည်မှာ Linux kernel နှင့် ကုမ္ပဏီ သို့မဟုတ် အသုံးပြုသူအဖွဲ့အစည်း (community) တစ်ခုမှ ထိန်းသိမ်းထားသော application များစွာကို စုစည်းထားသည့် အစုအဝေး (bundle) တစ်ခုဖြစ်သည်။ Distribution တစ်ခု၏ ရည်မှန်းချက်မှာ သတ်မှတ်ထားသော အသုံးပြုမှုပုံစံ သို့မဟုတ် အသုံးပြုသူအုပ်စုအတွက် operating system ပေါ်တွင် အလုပ်လုပ်မည့် kernel နှင့် application များကို အကောင်းဆုံးဖြစ်အောင် ညှိနှိုင်းပေးရန်ဖြစ်သည်။ Distribution များတွင် software ထည့်သွင်းခြင်းနှင့် စနစ်စီမံခန့်ခွဲခြင်း (system administration) တို့အတွက် သီးသန့် tool များ ပါဝင်လေ့ရှိသည်။ ထို့ကြောင့် အချို့ distribution များကို အသုံးပြုရလွယ်ကူရန် လိုအပ်သည့် desktop ပတ်ဝန်းကျင်များအတွက် အဓိကထားသုံးကြပြီး အချို့ကိုမူ ရရှိနိုင်သော အရင်းအမြစ်များကို အထိရောက်ဆုံး အသုံးပြုနိုင်ရန် server များပေါ်တွင် အဓိကထား အသုံးပြုကြသည်။

Distribution များကို ၎င်းတို့၏ မိသားစုအုပ်စု (distribution family) အလိုက် ခွဲခြားနိုင်သည်။ Debian မိသားစုဝင် distribution များသည် operating system ပေါ်တွင် software များ စီမံခန့်ခွဲရန် dpkg အမည်ရှိ package manager ကို အသုံးပြုကြသည်။ ထို package manager ဖြင့် ထည့်သွင်းနိုင်သော package များကို distribution community ၏ စေတနာ့ဝန်ထမ်းများက ထိန်းသိမ်းပေးထားသည်။ Package ဆိုသည်မှာ software နှင့် ၎င်းနှင့်သက်ဆိုင်သော configuration နှင့် documentation များ စုစည်းထားသော အစုအဝေးဖြစ်ပြီး အသုံးပြုသူအတွက် software ကို ထည့်သွင်းခြင်း၊ update ပြုလုပ်ခြင်းနှင့် အသုံးပြုခြင်းတို့ကို ပိုမိုလွယ်ကူစေသည်။

Debian GNU/Linux distribution သည် Debian မိသားစုတွင် အကြီးဆုံးဖြစ်သည်။ Debian GNU/Linux Project ကို ၁၉၉၃ ခုနှစ်တွင် Ian Murdock က စတင်ခဲ့သည်။ ယနေ့တွင် ထောင်ပေါင်းများစွာသော စေတနာ့ဝန်ထမ်းများက ဤ project တွင် ပါဝင်ဆောင်ရွက်နေကြသည်။ Debian GNU/Linux သည် အလွန်စိတ်ချရသော operating system တစ်ခုဖြစ်ရန် ရည်ရွယ်သည်။ ထို့အပြင် software ကို အသုံးပြုခြင်း၊ လေ့လာခြင်း၊ ဖြန့်ဝေခြင်းနှင့် မြှင့်တင်ခြင်းတို့တွင် အသုံးပြုသူ၏ လွတ်လပ်ခွင့်ကို လေးစားသော operating system တစ်ခုဖြစ်ရမည်ဟူသည့် Richard Stallman ၏ မျှော်မှန်းချက်ကိုလည်း အားပေးမြှင့်တင်သည်။ ထို့ကြောင့် ၎င်းတွင် proprietary software (မူပိုင်ခွင့်ကန့်သတ်ထားသော software) များကို ပုံသေသတ်မှတ်ချက် (default) အနေဖြင့် ထည့်သွင်းပေးထားခြင်း မရှိပါ။

### Embedded Systems (မြှုပ်နှံထားသော စနစ်များ)

Linux ကို စမတ်ဖုန်းများ၊ router များ၊ ဖျော်ဖြေရေးစနစ်များ (entertainment systems) နှင့် ကားများ၏ ထိန်းချုပ်မှုစနစ်များကဲ့သို့သော embedded device များတွင်လည်း အကျယ်တဝင့် အသုံးပြုကြသည်။ Embedded system တစ်ခုသည် ပိုမိုကြီးမားသော စနစ်တစ်ခု၏ အစိတ်အပိုင်းတစ်ခုအဖြစ် ပါဝင်နေသော ကွန်ပျူတာစနစ်တစ်ခုဖြစ်သည်။ ၎င်းကို သတ်မှတ်ထားသော လုပ်ဆောင်ချက်တစ်ခု သို့မဟုတ် အနည်းငယ်ကိုသာ လုပ်ဆောင်ရန် ဒီဇိုင်းထုတ်ထားခြင်းဖြစ်ပြီး အများအားဖြင့် အချိန်နှင့်တစ်ပြေးညီ လုပ်ဆောင်နိုင်စွမ်း (real-time computing) ကန့်သတ်ချက်များ ရှိတတ်သည်။

ဤစက်ပစ္စည်းများတွင် အရင်းအမြစ်များ (ဥပမာ- memory နှင့် storage) ကန့်သတ်ချက်ရှိလေ့ရှိသောကြောင့် Linux ၏ ပြောင်းလွယ်ပြင်လွယ်ရှိမှုနှင့် kernel ကို စိတ်ကြိုက်ပြင်ဆင်နိုင်မှုတို့က ၎င်းကို embedded application များအတွက် အကောင်းဆုံးရွေးချယ်စရာ ဖြစ်စေသည်။ Android သည် Linux kernel ကို အခြေခံထားသော လူသိအများဆုံး embedded system တစ်ခုဖြစ်ပြီး ကမ္ဘာတစ်ဝန်းရှိ သန်းပေါင်းများစွာသော စမတ်ဖုန်းများနှင့် tablet များတွင် အသုံးပြုလျက်ရှိသည်။

အခြားဥပမာတစ်ခုမှာ Raspberry Pi ဖြစ်သည်။ ၎င်းသည် ပညာရေးအတွက် ရည်ရွယ်ထုတ်လုပ်ထားသော သေးငယ်သည့် single-board computer တစ်ခုဖြစ်သော်လည်း embedded project များတွင်လည်း အသုံးများသည်။ Raspberry Pi အတွက် အဓိက operating system မှာ Raspbian (ယခုအခါ Raspberry Pi OS ဟုခေါ်သည်) ဖြစ်ပြီး ၎င်းသည် Debian ကို အခြေခံထားသော distribution တစ်ခုဖြစ်သည်။

### Linux and the Cloud (ကလောက်ပေါ်ရှိ Linux)

Cloud computing (ကလောက်ကွန်ပျူတာစနစ်) ဟူသည့် ဝေါဟာရသည် ကွန်ပျူတာ အရင်းအမြစ်များကို အများပြည်သူသုံး cloud ဝန်ဆောင်မှုပေးသူထံမှ ဝယ်ယူ၍ဖြစ်စေ၊ သီးသန့် cloud (private cloud) အဖြစ် ကိုယ်တိုင်လည်ပတ်၍ဖြစ်စေ စံသတ်မှတ်ထားသော နည်းလမ်းတစ်ခုဖြင့် အသုံးပြုခြင်းကို ဆိုလိုသည်။ ၂၀၁၇ ခုနှစ် အစီရင်ခံစာများအရ အများပြည်သူသုံး cloud လုပ်ငန်းပမာဏ (workload) ၏ ၉၀ ရာခိုင်နှုန်းတွင် Linux ကို အသုံးပြုထားသည်။ Amazon Web Services (AWS) မှသည် Google Cloud Platform (GCP) အထိ cloud ဝန်ဆောင်မှုပေးသူတိုင်းသည် Linux ပုံစံအမျိုးမျိုးကို ဝန်ဆောင်မှုပေးကြသည်။ ယနေ့ခေတ်တွင် Microsoft ပင်လျှင် ၎င်းတို့၏ Azure cloud ၌ Linux အခြေခံ virtual machine များကို ဝန်ဆောင်မှုပေးလျက်ရှိသည်။

Linux ကို များသောအားဖြင့် Infrastructure as a Service (IaaS) ဝန်ဆောင်မှု၏ အစိတ်အပိုင်းတစ်ခုအဖြစ် ပံ့ပိုးပေးလေ့ရှိသည်။ IaaS instance များသည် cloud အတွင်း မိနစ်ပိုင်းအတွင်း အသင့်အသုံးပြုနိုင်ရန် ဖန်တီးပေးထားသော virtual machine များဖြစ်သည်။ IaaS instance တစ်ခုကို စတင်အသုံးပြုသည့်အခါ data များပါဝင်သည့် image တစ်ခုကို ရွေးချယ်ရသည်။
