### နိဒါန်း (Introduction)
Terminal ပေါ်တွင် အလုပ်လုပ်ခြင်းသည် Linux စနစ်တစ်ခုကို ထိရောက်စွာ အသုံးပြုနိုင်ရန်အတွက် အခြေခံအကျဆုံးနှင့် အရေးကြီးဆုံး အရည်အချင်းတစ်ခု ဖြစ်သည်။ ဤသင်ခန်းစာတွင် Command Line ၏ တည်ဆောက်ပုံ၊ Command များ၏ ဗီဇနှင့် အခြေခံအသုံးအနှုန်းများကို လေ့လာသွားမည် ဖြစ်သည်။

Linux ၏ Command Line Interface (CLI) ကို Shell ဟုလည်း ခေါ်ဆိုကြပြီး၊ အသုံးအများဆုံး Shell မှာ Bash (Bourne Again Shell) ဖြစ်သည်။

### Command Line တည်ဆောက်ပုံ (Command Line Structure)
Command တစ်ခုကို ရိုက်ထည့်ရာတွင် အခြေခံအားဖြင့် အပိုင်း သုံးပိုင်း ပါဝင်လေ့ရှိသည် -
1. **Command:** လုပ်ဆောင်လိုသည့် ပရိုဂရမ် အမည် (ဥပမာ - `ls`)
2. **Options:** Command ၏ လုပ်ဆောင်ပုံကို ပြောင်းလဲပေးသည့် အရာများ (ဥပမာ - `-l`, `--help`)
3. **Arguments:** Command က လုပ်ဆောင်ရမည့် အရာ သို့မဟုတ် ပစ်မှတ် (ဥပမာ - ဖိုင်အမည် သို့မဟုတ် လမ်းကြောင်း)

ဥပမာ - `ls -l /etc` တွင် `ls` သည် command ဖြစ်ပြီး၊ `-l` သည် option ဖြစ်ကာ `/etc` သည် argument ဖြစ်သည်။

### Command များ၏ အမျိုးအစားများ (Command Behavior Types)
Command များသည် ၎င်းတို့၏ တည်ရှိပုံပေါ် မူတည်၍ အမျိုးအစား ကွဲပြားနိုင်သည် -
* **Built-in commands:** Shell ၏ အတွင်းပိုင်းတွင် တစ်ပါတည်း ပါဝင်ပြီးသား command များ။
* **External commands:** စနစ်အတွင်း သီးခြား ဖိုင်အဖြစ် တည်ရှိနေသော ပရိုဂရမ်များ။

မည်သည့် command သည် မည်သည့်အမျိုးအစား ဖြစ်သည်ကို သိရှိရန် `type` command ကို အသုံးပြု၍ စစ်ဆေးနိုင်သည်။

```bash
$ type cd
cd is a shell builtin
$ type ls
ls is aliased to `ls --color=auto'
```

### Quoting (ကိုးကားချက် သင်္ကေတများ အသုံးပြုခြင်း)
Shell တွင် အထူးသင်္ကေတများ (ဥပမာ - space, *, $) ၏ အဓိပ္ပာယ်ကို ထိန်းချုပ်ရန် Quoting ကို အသုံးပြုသည် -
* Double Quotes (`"`): အထူးသင်္ကေတ အများစု၏ အဓိပ္ပာယ်ကို ဖျောက်ပေးသော်လည်း `$` (variable) နှင့် `\` (escape) တို့ကိုမူ ဆက်လက် အလုပ်လုပ်စေသည်။
* Single Quotes (`'`): အတွင်းရှိ စာသားအားလုံးကို စာသားသက်သက် (literal string) အဖြစ်သာ မှတ်ယူပြီး မည်သည့် အထူးသင်္ကေတကိုမျှ အလုပ်မလုပ်စေပါ။
* Backslash (`\`): ၎င်းနောက်တွင် ကပ်လျက်ပါရှိသော သင်္ကေတတစ်ခုတည်း၏ အထူးအဓိပ္ပာယ်ကို ဖျောက်ပေးသည်။

### နိဒါန်း (Introduction)
ဤသင်ခန်းစာတွင် Shell အတွင်းရှိ Variables (ပြောင်းလဲကိန်းများ) ကို မည်သို့ အသုံးပြုရမည်နှင့် ၎င်းတို့ကို မည်သို့ စီမံခန့်ခွဲရမည်ကို အသေးစိတ် လေ့လာသွားမည် ဖြစ်သည်။

### Variables (ပြောင်းလဲကိန်းများ)
Variable ဆိုသည်မှာ အချက်အလက်များကို သိမ်းဆည်းထားရန် အသုံးပြုသော အမည်ပေးထားသည့် နေရာတစ်ခု ဖြစ်သည်။ Shell တွင် Variable များကို အဓိကအားဖြင့် နှစ်မျိုး ခွဲခြားနိုင်သည် -

1.  **Local Variables:** လက်ရှိ Shell session တစ်ခုတည်းတွင်သာ အသုံးပြုနိုင်သော variable များ။
2.  **Environment Variables:** လက်ရှိ Shell မှတစ်ဆင့် ဖွင့်လိုက်သော အခြား program များ သို့မဟုတ် shell များတွင်ပါ အသုံးပြုနိုင်သော variable များ။

Variable တစ်ခုကို တန်ဖိုးသတ်မှတ်ရန် `=` သင်္ကေတကို အသုံးပြုပြီး၊ ၎င်း၏ တန်ဖိုးကို ပြန်လည် ထုတ်ယူရန် `$` သင်္ကေတကို အသုံးပြုသည်။

```bash
$MY_NAME="Aung Aung"$ echo $MY_NAME
Aung Aung
```

### Variable များကို စီမံခန့်ခွဲခြင်း (Manipulating Variables)
* export: Local variable တစ်ခုကို Environment variable အဖြစ်သို့ ပြောင်းလဲရန် အသုံးပြုသည်။ သို့မှသာ အခြား program များက ၎င်း variable ကို သိရှိနိုင်မည် ဖြစ်သည်။
```bash
$ export MY_NAME
```
* unset: သတ်မှတ်ထားသော variable တစ်ခုကို ပြန်လည် ဖျက်ထုတ်ရန် အသုံးပြုသည်။
```bash
$ unset MY_NAME
```

### အရေးကြီးသော Environment Variables များ
စနစ်အတွင်းတွင် ကြိုတင်သတ်မှတ်ထားသော အရေးကြီးသည့် Environment variable အချို့ ရှိသည် -
* `PATH`: စနစ်မှ command များကို ရှာဖွေရမည့် လမ်းကြောင်းများစာရင်း။
* `HOME`: လက်ရှိအသုံးပြုသူ၏ Home directory လမ်းကြောင်း။
* `USER`: လက်ရှိ login ဝင်ထားသော အသုံးပြုသူ အမည်။
* `SHELL`: လက်ရှိ အသုံးပြုနေသော shell အမျိုးအစား လမ်းကြောင်း။

### History Command
history command ကို အသုံးပြု၍ ယခင်က ရိုက်ထည့်ခဲ့ဖူးသော command များ၏ စာရင်းကို ပြန်လည် ကြည့်ရှုနိုင်သည်။ ၎င်းသည် ရှည်လျားသော command များကို ပြန်လည် အသုံးပြုလိုသည့်အခါ အလွန် အသုံးဝင်သည်။
