### နိဒါန်း (Introduction)
Command line သည် အလွန်ရှုပ်ထွေးသော tool တစ်ခုဖြစ်သည်။ Command တစ်ခုချင်းစီတွင် ကိုယ်ပိုင်သီးသန့် option များရှိကြသောကြောင့် Linux system နှင့် အလုပ်လုပ်ရာတွင် documentation (လက်စွဲစာစောင်) သည် အလွန်အရေးကြီးပါသည်။ Documentation အများစုကို သိမ်းဆည်းထားသည့် `/usr/share/doc/` directory အပြင်၊ အခြားသော tool အမျိုးမျိုးကလည်း Linux command များအသုံးပြုပုံနှင့် ပတ်သက်၍ အချက်အလက်များကို ပံ့ပိုးပေးပါသည်။ ဤသင်ခန်းစာသည် အကူအညီရယူရန်အတွက် ထို documentation များကို ရှာဖွေကြည့်ရှုနိုင်မည့် နည်းလမ်းများအပေါ် အဓိကထားပါသည်။

Linux command line အတွင်း အကူအညီရယူနိုင်သော နည်းလမ်းများစွာ ရှိပါသည်။ `man`၊ `help` နှင့် `info` တို့မှာ ၎င်းတို့ထဲမှ အချို့ဖြစ်သည်။ Linux Essentials အတွက်မူ အကူအညီရယူရန် အသုံးအများဆုံး tool များဖြစ်သည့် `man` နှင့် `info` တို့ကို အဓိကထား လေ့လာသွားမည်ဖြစ်ပါသည်။ ဤသင်ခန်းစာ၏ အခြားခေါင်းစဉ်တစ်ခုမှာ ဖိုင်များကို ရှာဖွေခြင်း (locating files) ဖြစ်ပြီး၊ သင်သည် အဓိကအားဖြင့် `locate` command နှင့် အလုပ်လုပ်ရမည် ဖြစ်သည်။

### Command Line ပေါ်တွင် အကူအညီရယူခြင်း (Getting Help on the Command Line)

#### Built-in Help
Command အများစုကို `--help` parameter နှင့်အတူ ရိုက်နှိပ်ပါက ၎င်းတို့ကို မည်သို့အသုံးပြုရမည်ဆိုသည့် အကျဉ်းချုပ် ညွှန်ကြားချက်များကို ပြသပေးပါသည်။ Command အားလုံးတွင် ဤ switch မပါဝင်သော်လည်း၊ command တစ်ခု၏ parameter များအကြောင်းကို စတင်လေ့လာရန်အတွက် ၎င်းသည် ပထမဆုံး စမ်းသပ်ကြည့်သင့်သော နည်းလမ်းကောင်းတစ်ခု ဖြစ်သည်။ `--help` မှပေးသော ညွှန်ကြားချက်များသည် ဤသင်ခန်းစာတွင် ဆက်လက်ဆွေးနွေးမည့် အခြား documentation အရင်းအမြစ်များနှင့် နှိုင်းယှဉ်ပါက များသောအားဖြင့် အနည်းငယ်သာ ပါဝင်တတ်သည်ကို သတိပြုပါ။

#### Man Pages
Command အများစုတွင် manual page သို့မဟုတ် “man” page များ ပါရှိကြသည်။ ဤ documentation များကို များသောအားဖြင့် software နှင့်အတူ တစ်ပါတည်း ထည့်သွင်းပေးထားပြီး `man` command ဖြင့် ကြည့်ရှုနိုင်ပါသည်။ ကြည့်ရှုလိုသော command အမည်ကို `man` ၏ နောက်တွင် argument အဖြစ် ထည့်သွင်းပေးရပါသည် -

`$ man mkdir`

ဤ command သည် `mkdir` အတွက် man page ကို ဖွင့်ပေးမည်ဖြစ်သည်။ Man page အတွင်း သွားလာရန်အတွက် အပေါ်/အောက် မြှားခလုတ်များ သို့မဟုတ် space bar ကို အသုံးပြုနိုင်ပါသည်။ Man page မှ ပြန်ထွက်ရန် `Q` ခလုတ်ကို နှိပ်ပါ။

Man page တစ်ခုစီကို အပိုင်း (section) ၁၁ ခုအထိ အများဆုံး ခွဲခြားထားသော်လည်း အဆိုပါအပိုင်း အများစုမှာ ရွေးချယ်နိုင်သော (optional) အပိုင်းများသာ ဖြစ်သည် -

| အပိုင်း (Section) | ဖော်ပြချက် (Description) |
| :--- | :--- |
| NAME | Command အမည်နှင့် အကျဉ်းချုပ် ဖော်ပြချက် |
| SYNOPSIS | Command ၏ syntax (ရေးထုံး) ဖော်ပြချက် |
| DESCRIPTION | Command ၏ လုပ်ဆောင်ချက်များကို ဖော်ပြချက် |
| OPTIONS | အသုံးပြုနိုင်သော option များ |
| ARGUMENTS | အသုံးပြုနိုင်သော argument များ |
| FILES | ဆက်စပ်နေသော ဖိုင်များ |
| EXAMPLES | Command line အသုံးပြုပုံ နမူနာများ |
| SEE ALSO | ဆက်စပ်ခေါင်းစဉ်များသို့ ကိုးကားချက်များ |
| DIAGNOSTICS | သတိပေးချက်နှင့် Error message များ |
| COPYRIGHT | Command ကို ရေးသားသူ(များ) |
| BUGS | Command ၏ သိရှိထားပြီးသော ကန့်သတ်ချက်များ |

လက်တွေ့တွင် man page အများစု၌ ဤအပိုင်းအားလုံး ပါဝင်လေ့မရှိပါ။

---

Man page များကို အမျိုးအစား (category) ၈ ခုခွဲခြားထားပြီး ၎င်းတို့ကို ၁ မှ ၈ အထိ နံပါတ်စဉ် တပ်ထားပါသည်:

| အမျိုးအစား (Category) | ဖော်ပြချက် (Description) |
| :--- | :--- |
| ၁ | အသုံးပြုသူသုံး command များ (User commands)  |
| ၂ | System calls များ  |
| ၃ | C library ၏ function များ  |
| ၄ | Driver များနှင့် device file များ  |
| ၅ | Configuration file များနှင့် file format များ  |
| ၆ | ဂိမ်းများ (Games)  |
| ၇ | အထွေထွေ (Miscellaneous)  |
| ၈ | စနစ်အုပ်ချုပ်သူသုံး command များ (System administrator commands)  |
| ၉ | Kernel functions (စံသတ်မှတ်ချက်မဟုတ်ပါ)  |

Man page တစ်ခုစီသည် အမျိုးအစား တစ်ခုတည်းတွင်သာ တည်ရှိသော်လည်း၊ မတူညီသော အမျိုးအစားများတွင် အမည်တူ man page များ ရှိနေနိုင်ပါသည် ။ ဥပမာအားဖြင့် `passwd` command ကို ကြည့်နိုင်ပါသည်။ ဤ command သည် အသုံးပြုသူ၏ password ကို ပြောင်းလဲရန် အသုံးပြုသောကြောင့် ၎င်း၏ man page သည် အမျိုးအစား ၁ တွင် ရှိသည် ။ ထို့အပြင် `/etc/passwd` ဟုခေါ်သော password database ဖိုင်အတွက်လည်း man page ရှိပြီး ၎င်း၏အမည်မှာလည်း `passwd` ပင် ဖြစ်သည် ။ ဤဖိုင်သည် configuration ဖိုင်တစ်ခု ဖြစ်သောကြောင့် အမျိုးအစား ၅ တွင် ပါဝင်ပါသည် ။

Man page တစ်ခုကို ကိုးကားရာတွင် မည်သည့် man page ဖြစ်သည်ကို ခွဲခြားနိုင်ရန် passwd(1) သို့မဟုတ် passwd(5) ဟု အမျိုးအစားနံပါတ်ကို ထည့်သွင်းဖော်ပြလေ့ရှိကြသည် ။ ပုံမှန်အားဖြင့် `man passwd` ဟု ရိုက်နှိပ်ပါက ပထမဆုံး တွေ့ရှိရသော man page (ဤနေရာတွင် `passwd(1)`) ကိုသာ ပြသပေးမည်ဖြစ်သည် ။ မိမိအလိုရှိသော အမျိုးအစားကို သတ်သတ်မှတ်မှတ် ကြည့်ရှုလိုပါက `man 1 passwd` သို့မဟုတ် `man 5 passwd` ဟု ရိုက်နှိပ်၍ ကြည့်ရှုနိုင်ပါသည် ။

Man page အတွင်း သွားလာပုံနှင့် command line သို့ ပြန်ထွက်ပုံတို့ကို ရှေ့တွင် ဆွေးနွေးခဲ့ပြီး ဖြစ်ပါသည် ။ Man command သည် documentation အကြောင်းအရာများကို ပြသရန်အတွက် `less` command ကို အတွင်းပိုင်းတွင် အသုံးပြုထားပါသည် ။ `less` သည် man page အတွင်း စာသားများကို ရှာဖွေနိုင်ရန် ကူညီပေးသည် ။ ဥပမာ - "linux" ဟူသော စကားလုံးကို ရှာဖွေလိုပါက လက်ရှိရောက်ရှိနေသော နေရာမှ အောက်ဘက်သို့ ရှာရန် `/linux` ကို အသုံးပြုနိုင်ပြီး၊ အပေါ်ဘက်သို့ ပြန်ရှာရန် `?linux` ကို အသုံးပြုနိုင်ပါသည် ။ ဤသို့ရှာဖွေခြင်းသည် ကိုက်ညီသော စာသားများကို highlight ပြုလုပ်ပေးမည်ဖြစ်ပြီး ပထမဆုံးတွေ့ရှိသော နေရာသို့ စာမျက်နှာကို ရွှေ့ပေးမည်ဖြစ်သည် ။ နောက်ထပ် ကိုက်ညီသော စာသားကို ထပ်မံရှာဖွေလိုပါက `N` ကို နှိပ်နိုင်ပါသည် ။ ထပ်ဆောင်း လုပ်ဆောင်ချက်များအကြောင်း သိရှိလိုပါက `H` ကို နှိပ်၍ အချက်အလက်များ ပါဝင်သော menu ကို ကြည့်ရှုနိုင်ပါသည် ။

### Info Pages
Linux တွင် အလုပ်လုပ်နေစဉ် အကူအညီပေးနိုင်မည့် နောက်ထပ် tool တစ်ခုမှာ `info` ဖြစ်သည်။ ၎င်းသည် man page နှင့် ဆင်တူသော်လည်း man page ထက် ပိုမိုပြည့်စုံသော documentation များကို ပေးစွမ်းနိုင်ရန် ရည်ရွယ်၍ GNU project မှ ဖန်တီးထားခြင်း ဖြစ်သည်။

`info` စာမျက်နှာများကို hypertext document များကဲ့သို့ ဖွဲ့စည်းထားပြီး တစ်ခုနှင့်တစ်ခု ချိတ်ဆက်ထားသော link (links) များ ပါဝင်သည်။ အဆိုပါ link များကို ဤစနစ်တွင် "nodes" ဟု ခေါ်ဆိုသည်။

`info` ကို အသုံးပြုရန် -

`$ info mkdir`

`info` reader အတွင်း သွားလာရန် အခြေခံ command အချို့မှာ -

* **Up/Down Arrow Keys:** တစ်ကြောင်းချင်းစီ ရွှေ့ရန်။
* **Page Up/Down:** စာမျက်နှာအလိုက် ရွှေ့ရန်။
* **Tab:** နောက်ထပ် link (node) တစ်ခုသို့ ကူးပြောင်းရန်။
* **Enter:** ရွေးချယ်ထားသော link (node) ကို ဖွင့်ရန်။
* **N (Next):** နောက်ထပ် node သို့ သွားရန်။
* **P (Previous):** ယခင် node သို့ ပြန်သွားရန်။
* **U (Up):** လက်ရှိ node ၏ အထက်အဆင့် (Parent node) သို့ ပြန်သွားရန်။
* **Q (Quit):** `info` reader မှ ထွက်ရန်။

### /usr/share/doc/ Directory
Linux system ပေါ်တွင် ထည့်သွင်းထားသော software package တိုင်းအတွက် documentation များသည် များသောအားဖြင့် `/usr/share/doc/` အောက်တွင် ရှိကြပါသည်။ ဤ directory အတွင်းတွင် software တစ်ခုချင်းစီအတွက် သီးသန့် subdirectory များ ရှိပြီး၊ ၎င်းတို့ထဲတွင် README ဖိုင်များ၊ ဥပမာ configuration ဖိုင်များ (example files) နှင့် အခြားသော အထောက်အထားစာစောင်များ ပါဝင်လေ့ရှိပါသည်။

ဤဖိုင်များကို ကြည့်ရှုရန် `ls /usr/share/doc/` ဟု ရိုက်နှိပ်၍ ရှာဖွေနိုင်ပြီး `zless` သို့မဟုတ် `cat` ကဲ့သို့သော command များကို အသုံးပြု၍ ဖတ်ရှုနိုင်ပါသည်။

### ဖိုင်များကို ရှာဖွေခြင်း (Locating Files)
တစ်ခါတစ်ရံတွင် သင်သည် ဖိုင်တစ်ခု မည်သည့်နေရာတွင် ရှိနေသည်ကို ရှာဖွေရန် လိုအပ်နိုင်ပါသည်။ ထိုအခါ `locate` command ကို အသုံးပြုနိုင်ပါသည်။

`$ locate setup.log`

`locate` command သည် system တစ်ခုလုံးရှိ ဖိုင်အမည်များကို သိမ်းဆည်းထားသော database တစ်ခုအတွင်း၌ ရှာဖွေခြင်း ဖြစ်သောကြောင့် အလွန်မြန်ဆန်ပါသည်။ သို့သော် system အတွင်း အသစ်ထည့်လိုက်သော ဖိုင်များကို database update မဖြစ်သေးပါက ရှာတွေ့မည် မဟုတ်ပါ။ Database ကို update ပြုလုပ်ရန် system administrator (root) အခွင့်အရေးဖြင့် `updatedb` command ကို အသုံးပြုရပါသည်။

`find` command သည် `locate` ထက် ပိုမို အစွမ်းထက်ပြီး အသေးစိတ် ရှာဖွေနိုင်သော tool တစ်ခုဖြစ်သည်။ ၎င်းသည် database ကို အသုံးမပြုဘဲ ဖိုင်စနစ် (live filesystem) အတွင်း တိုက်ရိုက်ရှာဖွေသည်။ ထို့ကြောင့် locate ထက် ပိုမိုနှေးကွေးသော်လည်း အမြဲတမ်း တိကျသော ရလဒ်ကို ပေးသည်။ ဖိုင်များကို criteria အမျိုးမျိုး (ဥပမာ - အမည်၊ အရွယ်အစား၊ ပြင်ဆင်ခဲ့သည့်အချိန်) ပေါ်မူတည်၍ ရှာဖွေပေးနိုင်သော အစွမ်းထက်သည့် command တစ်ခုဖြစ်သည်။
* **အခြေခံ တည်ဆောက်ပုံ:** `find [path] [parameter] [argument]`
* **ဥပမာများ:**
    * လက်ရှိ folder အောက်တွင် "myfile.txt" ကို ရှာရန် -
      `$ find . -name "myfile.txt"`
    * တစ်စနစ်လုံး (root directory) အောက်တွင် ရှာရန် -
      `$ find / -name "myfile.txt"`
    * အမည်အကြီးအသေး မခွဲခြားဘဲ ရှာရန် (`-iname`) -
      `$ find . -iname "MYFILE.txt"`

### အနှစ်ချုပ် (Summary)

ဤသင်ခန်းစာတွင် Linux command line ပေါ်၌ အကူအညီရယူနိုင်ရန်နှင့် ဖိုင်များကို ရှာဖွေနိုင်ရန်အတွက် အောက်ပါအချက်များကို လေ့လာခဲ့ပြီး ဖြစ်သည်-
* **Help Switch:** Command အများစုတွင် ပါဝင်သော `--help` option ကို အသုံးပြု၍ အခြေခံ syntax နှင့် option များကို အမြန်ကြည့်ရှုနိုင်ပါသည်။
* **Documentation အရင်းအမြစ်များ:** Linux တွင် အကူအညီရယူရန် `man` pages၊ `info` pages နှင့် `/usr/share/doc/` directory တို့ကို အသုံးပြုနိုင်ပါသည်။
* **Man Pages:** `man` command သည် အသုံးအများဆုံး documentation ဖြစ်ပြီး၊ ၎င်းတို့ကို section အမျိုးမျိုး (ဥပမာ- user commands အတွက် Section 1၊ configuration files အတွက် Section 5) ခွဲခြားထားပါသည်။ `less` reader ကို အသုံးပြု၍ စာသားများကို ရှာဖွေနိုင်ပါသည်။
* **Info Pages:** GNU project မှ ပံ့ပိုးထားသော ပိုမိုပြည့်စုံသည့် hypertext documentation အမျိုးအစား ဖြစ်ပြီး၊ link (nodes) များအကြား ကူးသန်းသွားလာနိုင်ပါသည်။
* **Locating Files:** `locate` command ကို အသုံးပြု၍ system database အတွင်းရှိ ဖိုင်များကို အလွန်မြန်ဆန်စွာ ရှာဖွေနိုင်ပါသည်။ Database အသစ်ဖြစ်စေရန် `updatedb` ကို အသုံးပြုရန် လိုအပ်ပါသည်။ သို့မဟုတ် `find` command အသုံးပြု၍ အသေးစိတ် ရှာနိုင်ပါသည်။
