### နိဒါန်း (Introduction)
Command Line သည် အလွန်ကျယ်ပြန့်သော tool တစ်ခုဖြစ်သည်။ command တစ်ခုစီတွင် ကိုယ်ပိုင် option များစွာရှိသောကြောင့် Linux စနစ်တွင် အလုပ်လုပ်သည့်အခါ documentation (စာရွက်စာတမ်း အထောက်အထားများ) သည် အလွန်အရေးကြီးသည်။ ဤသင်ခန်းစာတွင် Linux command များအကြောင်း အကူအညီရယူနိုင်သည့် နည်းလမ်းများနှင့် ဖိုင်များကို ရှာဖွေနည်းများကို လေ့လာပါမည်။

### Command Line တွင် အကူအညီရယူခြင်း (Getting Help on the Command Line)

#### Built-in Help
Command အများစုတွင် `--help` parameter ကို ထည့်သွင်းရိုက်နှက်ခြင်းဖြင့် အသုံးပြုပုံ အကျဉ်းချုပ်ကို ကြည့်ရှုနိုင်သည်။ ၎င်းသည် command တစ်ခု၏ အခြေခံ parameter များကို လျင်မြန်စွာ သိရှိရန် အကောင်းဆုံးနည်းလမ်းဖြစ်သည်။

#### Man Pages (Manual Pages)
Linux ရှိ command အားလုံးနီးပါးတွင် "man" page ဟုခေါ်သော လက်စွဲစာမျက်နှာများ ပါရှိသည်။ ၎င်းကို `man` command ဖြင့် ကြည့်ရှုနိုင်သည်။
ဥပမာ - `$ man mkdir`


Man page များကို အဓိက ကဏ္ဍ (Category) ၈ ခုခွဲထားသည် -
1. **User command:** အသုံးပြုသူသုံး command များ
2. **System calls:** စနစ်ခေါ်ဆိုမှုများ
5. **Configuration files:** စနစ်သတ်မှတ်ချက်ဖိုင်များနှင့် ဖိုင်ပုံစံများ
8. **System administrator commands:** စနစ်အုပ်ချုပ်သူသုံး command များ

#### Info Pages
Info pages များသည် man pages များထက် ပိုမိုအသေးစိတ်ပြီး Internet စာမျက်နှာများကဲ့သို့ hypertext ပုံစံဖြင့် တည်ဆောက်ထားသည်။ `info` command ကို အသုံးပြု၍ ကြည့်ရှုနိုင်သည်။

### ဖိုင်များကို ရှာဖွေခြင်း (Locating Files)

#### locate command
စနစ်အတွင်းရှိ ဖိုင်တစ်ခုကို အမြန်ဆုံးရှာဖွေနိုင်သည့် command မှာ `locate` ဖြစ်သည်။ ၎င်းသည် database အတွင်း ရှာဖွေခြင်းဖြစ်သောကြောင့် အလွန်မြန်ဆန်သည်။
ဥပမာ - `$ locate note`

#### find command
`find` သည် locate ထက် ပိုမိုစွမ်းဆောင်ရည်မြင့်မားသည်။ ၎င်းသည် database ကို အသုံးမပြုဘဲ directory များအတွင်း တိုက်ရိုက်ဝင်ရောက် ရှာဖွေသည်။ ထို့ကြောင့် အသစ်ဖန်တီးထားသော ဖိုင်များကို ရှာဖွေရာတွင် ပိုမိုတိကျသည်။
ဥပမာ - `$ find . -name thesis.pdf` (လက်ရှိ folder အတွင်း thesis.pdf အမည်ရှိဖိုင်ကို ရှာဖွေခြင်း)
