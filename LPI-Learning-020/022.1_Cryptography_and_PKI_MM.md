# 022.1 Cryptography and Public Key Infrastructure (PKI)
## (ကုဒ်လျှို့ဝှက်ရေးနှင့် PKI အခြေခံ)

---

## အကျဉ်းချုပ်
ဒီအပိုင်းမှာ hashing, symmetric/asymmetric encryption, hybrid cryptography, PFS, PKI, X.509 certificates, CA/Root CA, chain of trust, Let's Encrypt စတာတွေကို အခြေခံမှ ရှင်းပြထားပါတယ်။

## Hash Functions
- Input တစ်ခုကို fixed-length digest အဖြစ် ပြောင်းပေး
- Feature: input နည်းနည်းပြောင်းတာနဲ့ hash အရမ်းပြောင်း
- One-way ဖြစ်လို့ original ပြန်မထုတ်လွယ်
- Use cases:
  - File integrity verification (ဥပမာ SHA-256)
  - Password storage (hash + salt)
  - Digital signatures (message digest)
- MD5 က collision weakness ကြောင့် modern security မသုံးသင့်

---

## Symmetric Encryption
- Encrypt/Decrypt နှစ်ဖက်လုံး **တူညီတဲ့ key** သုံး
- အားသာချက်: မြန် (large data အတွက်ကောင်း)
- အားနည်းချက်: key distribution ခက်
- ဥပမာ algorithm: **AES**

---

## Asymmetric Encryption
- **Public key** (encrypt/verify) + **Private key** (decrypt/sign)
- အားသာချက်: secret key မမျှဝေဘဲ secure exchange
- အားနည်းချက်: နှေး (compute heavy)
- ဥပမာ: **RSA**, **ECC/ECDSA**

---

## Hybrid Cryptography
- Asymmetric ကို key exchange အတွက်သုံး၊ data encryption ကို symmetric သုံး
- HTTPS/TLS မှာ အဓိကသုံးတဲ့ model

---

## Perfect Forward Secrecy (PFS)
- Session တစ်ခုချင်းစီက ephemeral key သုံး
- Long-term key ပျက်သွားရင်တောင် အရင် session တွေကို ပြန်မဖော်နိုင်
- DH/ECDH တို့က session key စိုက်ထုတ်ရာမှာ အသုံးများ

---

## End-to-End vs Transport Encryption
- **E2EE**: sender↔receiver သာ decrypt လုပ်နိုင် (provider မဖတ်နိုင်)
- **Transport**: client↔server ကြားသာ encrypt (server မှာ decrypt ဖြစ်နိုင်)

---

## PKI (Public Key Infrastructure)
- Public key ကို “အမှန်ပိုင်ရှင်” နဲ့ link လုပ်ပြီး ယုံကြည်မှုတည်ဆောက်တဲ့ framework
- **CA (Certificate Authority)** က certificate issue/sign/revoke လုပ်
- **Root CA** ကို OS/browser ထဲ trust store နဲ့ တင်ထား

### Chain of Trust
- End-entity cert → Intermediate CA → Root CA
- Browser က chain ကို verify လုပ်ပြီးမှ secure connection တည်ဆောက်

---

## X.509 Certificates
- Standard certificate format
- အရေးကြီး field များ: Subject, Issuer, Validity, Public Key, Signature, Extensions
- CSR (Certificate Signing Request) တင်ပြီး CA က issue လုပ်

---

## Let’s Encrypt
- Free DV certificates
- Short validity (automation renewal အရေးကြီး)
