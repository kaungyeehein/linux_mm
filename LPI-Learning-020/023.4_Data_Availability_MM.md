# 023.4 Data Availability
## (ဒေတာရရှိနိုင်မှုနှင့် Backup များ)

---

## အဓိကအယူအဆ
- Availability ကို ထိန်းဖို့ backup/restore strategy လို
- Backup မရှိရင် ransomware/accidental delete/disaster မှာ recover မလွယ်

---

## Backup Importance
- Business continuity
- Recovery from corruption/attack

---

## Backup Types
- **Full**: အကုန် backup (time/space အများ)
- **Incremental**: နောက်ဆုံး backup ကနေပြောင်းတာပဲ (space efficient, restore complex)
- **Differential**: နောက်ဆုံး full ကနေပြောင်းတာ (restore simpler)
- **Snapshot**: point-in-time state

---

## Backup Security
- Backup ကို encrypt
- Access control + audit
- Offsite/immutable/air-gapped backups (ransomware resilience)
- Regular restore testing

---

## Cloud Storage Considerations
- Sync ≠ Backup (delete က sync ဖြစ်နိုင်)
- Internet dependency
- Sharing permissions အန္တရာယ်

---

## Exam Focus
- Full vs Incremental vs Differential
- Backup security (encrypt + access control)
- Cloud sync risk

---

## Quick Check
1. Incremental backup restore အတွက် ဘာတွေလို?
2. Cloud sync ကို backup လို့ မထင်သင့်တာ ဘာကြောင့်?
