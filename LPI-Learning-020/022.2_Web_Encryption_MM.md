# 022.2 Web Encryption
## (Web လုံခြုံရေး: HTTPS/TLS နှင့် Certificate Validation)

---

## Plain HTTP vs HTTPS
- **HTTP**
  - Plain text
  - Eavesdropping / tampering အလွယ်
  - Default port: 80
- **HTTPS**
  - HTTP over TLS
  - Confidentiality + Integrity + Authentication
  - Default port: 443

---

## TLS အခြေခံ
- TLS handshake
  - Server identity verify (certificate)
  - Key exchange
  - Session encryption setup
- TLS record
  - Symmetric encryption + integrity check (MAC)

---

## Certificate Fields (HTTPS အတွက် အရေးကြီး)
- Subject / Common Name
- **subjectAltName (SAN)**: DNS names / IP addresses
- Validity (notBefore / notAfter)
- Issuer (CA)

---

## Website Identity Verification (Domain Match)
- SAN DNS name ရှိရင် SAN ကိုပဲ အသုံးပြု
- Wildcard `*.example.com` သည် single subdomain level ကိုသာ match လုပ်
- IP URL သုံးလျှင် SAN ထဲက IP Address field နဲ့ exact match ဖြစ်ရ

---

## Browser Validity Checks
- Domain/SAN match
- Chain of trust (Intermediate → Root)
- Validity period (expired/not yet valid?)
- Revocation check (CRL/OCSP)
- Signature verify (tampering?)
- Key usage/extensions (purpose mismatch?)

---

## User Side: Encrypted Website စစ်နည်း
- Address bar **lock icon**
- URL `https://`
- Warning messages:
  - “Not Secure” (HTTP)
  - “Your connection is not private” (expired/untrusted/misconfigured)

---

## Exam Focus
- HTTP vs HTTPS
- TLS role
- Port 80 vs 443
- SAN, wildcard rules
- Browser validation checklist

---

## Quick Check
1. HTTPS မှာ server identity ဘယ်လို verify လဲ?
2. Certificate expired ဖြစ်ရင် ဘာဖြစ်မလဲ?
3. `*.example.com` က `a.b.example.com` ကို match လုပ်နိုင်လား?
