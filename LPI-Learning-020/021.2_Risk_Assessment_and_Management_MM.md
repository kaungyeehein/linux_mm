# 021.2 Risk Assessment and Management
## (အန္တရာယ်အကဲဖြတ်ခြင်းနှင့် စီမံခန့်ခွဲခြင်း)

---

## အဓိကအယူအဆ
- Vulnerability တစ်ခုတွေ့ရုံနဲ့ “အရေးအကြီးဆုံး” မဟုတ်နိုင်
- **Risk = Likelihood (ဖြစ်နိုင်ချေ) × Impact (နစ်နာမှု)** ကို စဉ်းစားပြီး ဦးစားပေးတုံ့ပြန်ရမယ်

---

## Security Information Sources (သတင်းအချက်အလက်ရင်းမြစ်)
- **CVE (Common Vulnerabilities and Exposures)**
  - Vulnerability များကို standard ID (CVE-ID) နဲ့ သတ်မှတ်
  - Patch/mitigation ကို track လုပ်ရာမှာ အရေးကြီး
- **CERT (Computer Emergency Response Team)**
  - Incident response, advisories, threat intel ပေး

---

## Incident Classification & Vulnerability Types
- **Zero-day**: Vendor မသိသေး/patch မထွက်သေးတဲ့ flaw
- **Remote Code Execution (RCE)**: Remote ကနေ code run လုပ်နိုင်
- **Privilege Escalation**: အခွင့်အရေးမြှင့်ယူခြင်း (vertical/horizontal)
- **Untargeted attacks**: automation နဲ့ random targets ကို တိုက်
- **APT (Advanced Persistent Threat)**: Targeted, stealthy, long-term intrusion

---

## Security Assessments
- **Vulnerability Scanning**
  - known flaws ရှိ/မရှိ စစ်ခြင်း (report သဘော)
- **Penetration Testing**
  - တကယ်ဝင်ဖောက်ကြည့်ပြီး exploit လုပ်နိုင်မှုကို စမ်းခြင်း (ပိုမိုပြင်းထန်)

---

## IT Forensics (ဒီဂျစ်တယ်ဖိုရင်ဆစ်)
- Incident ဖြစ်ပြီးနောက် evidence စုဆောင်း/ထိန်းသိမ်း/ခွဲခြမ်း
- Goal: ဘယ်လိုဖြစ်လဲ၊ ဘယ်အရာတွေထိခိုက်လဲ၊ ပြန်မဖြစ်အောင် ဘာလုပ်မလဲ

---

## ISMS & Incident Response
- **ISMS (Information Security Management System)**
  - Policies + process + people + tech controls ကို တပြိုင်နက်စီမံ
  - Asset inventory, access control, testing plan, acceptable use စတဲ့ မူဝါဒ
- **IRP (Incident Response Plan)**
  - Detect → Analyze → Contain → Eradicate → Recover → Lessons learned
- **ISIRT (Incident Response Team)**
  - Tech + Legal + Comms + Management ပူးပေါင်း

---

## Exam Focus
- CVE / CVE-ID ကို သိ
- Scanning vs Pen-testing ကွာခြားချက်
- Zero-day / RCE / Privilege escalation / APT
- ISMS, IRP, ISIRT အဓိပ္ပါယ်

---

3. IRP အဆင့်တွေကို စဉ်တကျ ပြောနိုင်လား?
