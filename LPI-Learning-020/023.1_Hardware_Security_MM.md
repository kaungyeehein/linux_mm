# 023.1 Hardware Security
## (ဟာ့ဒ်ဝဲ လုံခြုံရေး)

---

## အဓိကအယူအဆ
- Physical access ရသွားရင် security controls အများစုကို bypass လုပ်နိုင်
- IoT/Smart devices တိုးလာသလို hardware-level security အရေးကြီးလာသည်

---

## Physical Access Risks
- Device ကို ခိုးယူ/ဖောက်ထွင်း → data theft
- BIOS/UEFI settings ပြောင်း, boot media သုံးပြီး attack
- Mitigation: FDE, BIOS password, secure boot, device lock

---

## USB Security
- Malware injection, data exfiltration, badUSB
- Mitigation: disable unused ports, allowlist devices, scan, encryption

---

## Bluetooth Security
- Unauthorized pairing, data theft, spoofing
- Mitigation: turn off when unused, secure pairing, updates

---

## RFID Security
- Eavesdropping, cloning, skimming
- Mitigation: RFID shield, strong protocols, monitoring

---

## Trusted Computing (TPM)
- Key storage, secure boot, attestation
- Integrity checking အတွက် အရေးကြီး

---

## Exam Focus
- Physical access = high risk
- USB/Bluetooth/RFID threats
- TPM/Trusted computing concept

---
