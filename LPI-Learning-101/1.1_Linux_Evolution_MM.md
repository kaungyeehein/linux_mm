### နိဒါန်း (Introduction)
Linux သည် လူကြိုက်အများဆုံး operating system များထဲမှ တစ်ခုဖြစ်သည်။ ၎င်းကို ၁၉၉၁ ခုနှစ်တွင် Linus Torvalds မှ စတင်ဖော်ဆောင်ခဲ့ခြင်းဖြစ်သည်။ ဤ operating system သည် ၁၉၇၀ ပြည့်လွန်နှစ်များအတွင်း AT&T ဓာတ်ခွဲခန်းမှ ဖော်ဆောင်ခဲ့သော Unix အမည်ရှိ အခြား operating system တစ်ခုကို အခြေခံ၍ စိတ်ကူးရရှိခဲ့ခြင်း ဖြစ်သည်။ Unix သည် အသေးစား ကွန်ပျူတာများအတွက် ရည်ရွယ်ခဲ့သည်။ ထိုအချိန်က "အသေးစား" ကွန်ပျူတာဆိုသည်မှာ လေအေးပေးစက်ပါသော ခန်းမတစ်ခုလုံးစာ နေရာမယူဘဲ၊ တန်ဖိုးအားဖြင့် ဒေါ်လာတစ်သန်းအောက်ရှိသည့် စက်များကို ဆိုလိုခြင်းဖြစ်သည်။ နောက်ပိုင်းတွင် လူနှစ်ဦး မနိုင်သည့် စက်များဟု သတ်မှတ်ခဲ့ကြသည်။ ထိုအချိန်တွင် ရုံးသုံးကွန်ပျူတာများကဲ့သို့သော x86 platform ကို အခြေခံသည့် ကွန်ပျူတာများပေါ်တွင် ဈေးနှုန်းသက်သာသော Unix စနစ်တစ်ခုမှာ အလွယ်တကူ မရရှိနိုင်သေးပေ။ ထို့ကြောင့် ထိုစဉ်က ကျောင်းသားတစ်ဦးဖြစ်သည့် Linus သည် ဤ platform ပေါ်တွင် အသုံးပြုနိုင်မည့် Unix နှင့် ဆင်တူသော operating system တစ်ခုကို စတင်အကောင်အထည်ဖော်ခဲ့ခြင်း ဖြစ်သည်။

အများအားဖြင့် Linux သည် Unix ၏ အခြေခံမူများနှင့် စိတ်ကူးစိတ်သန်းများကို အသုံးပြုထားသော်လည်း၊ Linux ကိုယ်တိုင်သည် သီးခြားစီမံကိန်းတစ်ခုဖြစ်သောကြောင့် Unix code များ ပါဝင်ခြင်းမရှိပါ။ Linux ကို ကုမ္ပဏီတစ်ခုတည်းက ပံ့ပိုးထားခြင်းမျိုးမဟုတ်ဘဲ နိုင်ငံတကာမှ ပရိုဂရမ်မာများ စုစည်းထားသည့် အဖွဲ့အစည်း (community) မှ ပံ့ပိုးထားခြင်းဖြစ်သည်။ ၎င်းကို အခမဲ့ရရှိနိုင်သောကြောင့် မည်သူမဆို ကန့်သတ်ချက်မရှိဘဲ အသုံးပြုနိုင်သည်။

### Distributions (Linux ဖြန့်ချိမှုပုံစံများ)

Linux distribution ဆိုသည်မှာ ကုမ္ပဏီတစ်ခု သို့မဟုတ် အသုံးပြုသူအဖွဲ့အစည်း (community) တစ်ခုမှ စီမံထိန်းသိမ်းထားသော Linux kernel နှင့် ရွေးချယ်ထားသော application များ ပေါင်းစည်းထားသည့် အစုအဝေးတစ်ခုဖြစ်သည်။ distribution တစ်ခု၏ ရည်မှန်းချက်မှာ သတ်မှတ်ထားသော အသုံးပြုမှုပုံစံ (use case) သို့မဟုတ် အသုံးပြုသူအုပ်စုအတွက် operating system ပေါ်တွင် အလုပ်လုပ်မည့် kernel နှင့် application များကို အကောင်းဆုံးဖြစ်အောင် ညှိနှိုင်းပေးရန်ဖြစ်သည်။ distribution များတွင် software ထည့်သွင်းခြင်းနှင့် system စီမံခန့်ခွဲခြင်းအတွက် သီးသန့် tool များ ပါဝင်လေ့ရှိသည်။ ထို့ကြောင့် အချို့သော distribution များကို အသုံးပြုရလွယ်ကူရန် လိုအပ်သည့် desktop ပတ်ဝန်းကျင်များအတွက် အဓိကအသုံးပြုကြပြီး၊ အချို့ကိုမူ ရရှိနိုင်သော အရင်းအမြစ်များကို အထိရောက်ဆုံး အသုံးချနိုင်ရန် server များပေါ်တွင် အဓိကအသုံးပြုကြသည်။

Distribution များကို ၎င်းတို့ပိုင်ဆိုင်သည့် distribution family (မျိုးနွယ်စု) အလိုက်လည်း ခွဲခြားနိုင်သည်။ Debian မျိုးနွယ်စုဝင် distribution များသည် operating system ပေါ်တွင် အလုပ်လုပ်သော software များကို စီမံခန့်ခွဲရန် `dpkg` ဟုခေါ်သော package manager ကို အသုံးပြုကြသည်။ ထည့်သွင်းနိုင်သော package များကို community အဖွဲ့ဝင်များက စေတနာအလျောက် ထိန်းသိမ်းပေးကြသည်။ ၎င်းတို့သည် software ကို operating system ပေါ်တွင် မည်သို့ထည့်သွင်းရမည်နှင့် မည်သို့ configure လုပ်ရမည်ကို သတ်မှတ်ရန် `.deb` package format ကို အသုံးပြုကြသည်။ package ဆိုသည်မှာ အသုံးပြုသူများ software ကို ထည့်သွင်းခြင်း၊ update လုပ်ခြင်းနှင့် အသုံးပြုခြင်းတို့တွင် ပိုမိုလွယ်ကူစေရန် software နှင့်အတူ သက်ဆိုင်ရာ configuration နှင့် documentation များပါဝင်သော အစုအဝေးတစ်ခုဖြစ်သည်။

Debian GNU/Linux distribution သည် Debian မျိုးနွယ်စုတွင် အကြီးမားဆုံးဖြစ်ပြီး ၁၉၉၃ ခုနှစ်တွင် Ian Murdock က စတင်ခဲ့သည်။ Debian သည် ယုံကြည်စိတ်ချရသော operating system တစ်ခုဖြစ်ရန် ရည်ရွယ်ပြီး software ကို လွတ်လပ်စွာ အသုံးပြုခွင့်၊ လေ့လာခွင့်နှင့် ပြုပြင်ခွင့်ပေးသော Richard Stallman ၏ အယူအဆကို ထောက်ခံသောကြောင့် proprietary (မူပိုင်ခွင့်ရှိသော) software များကို မူလအားဖြင့် ထည့်သွင်းပေးထားခြင်း မရှိပါ။

အခြားသတိပြုသင့်သော Debian အခြေခံ distribution တစ်ခုမှာ Ubuntu ဖြစ်သည်။ ၎င်းကို ၂၀၀၄ ခုနှစ်တွင် Mark Shuttleworth နှင့် အဖွဲ့က ဖန်တီးခဲ့ပြီး၊ လွယ်ကူသော Linux desktop ပတ်ဝန်းကျင်တစ်ခု ဖြစ်လာစေရန် ရည်ရွယ်သည်။ Ubuntu သည် ၆ လတစ်ကြိမ် ဗားရှင်းအသစ်များ ထုတ်ဝေလေ့ရှိပြီး ၂ နှစ်တစ်ကြိမ်တွင် ရေရှည်ထောက်ပံ့မှုပေးသော Long-term Support (LTS) ဗားရှင်းကို ထုတ်ဝေသည်။

Red Hat သည် ၂၀၁၉ ခုနှစ်တွင် IBM မှ ဝယ်ယူခဲ့သော Red Hat ကုမ္ပဏီမှ ထုတ်လုပ်သည့် distribution ဖြစ်ပြီး Red Hat Enterprise Linux (RHEL) ဟု လူသိများသည်။ ၎င်းသည် လုပ်ငန်းခွင် server များအတွက် ယုံကြည်စိတ်ချရသော ဖြေရှင်းချက်အဖြစ် ရည်ရွယ်ပြီး အချို့သော အစိတ်အပိုင်းများအတွက် အခကြေးငွေပေးရသော subscription များ လိုအပ်သည်။ CentOS စီမံကိန်းသည် RHEL ၏ အခမဲ့ရရှိနိုင်သော source code များကို အသုံးပြု၍ ပြန်လည်တည်ဆောက်ထားခြင်းဖြစ်ပြီး commercial support မပါဝင်သော်လည်း အခမဲ့အသုံးပြုနိုင်သည်။ Fedora စီမံကိန်းသည် desktop ကွန်ပျူတာများအတွက် ရည်ရွယ်ပြီး နည်းပညာအသစ်များကို လျင်မြန်စွာ လက်ခံကျင့်သုံးသောကြောင့် RHEL အတွက် စမ်းသပ်ရာနေရာအဖြစ် ယူဆကြသည်။ Red Hat အခြေခံ distribution များအားလုံးသည် `.rpm` package format ကို အသုံးပြုကြသည်။

SUSE ကုမ္ပဏီကို ၁၉၉၂ ခုနှစ်တွင် ဂျာမနီနိုင်ငံ၌ စတင်ခဲ့သည်။ SUSE Linux သည် ၎င်း၏ YaST configuration tool ကြောင့် လူသိများပြီး ၎င်း tool ဖြင့် software၊ hardware၊ server နှင့် network များကို စီမံခန့်ခွဲနိုင်သည်။ SUSE သည်လည်း RHEL ကဲ့သို့ပင် စီးပွားရေးလုပ်ငန်းများအတွက် SUSE Linux Enterprise Server (SLES) ကို ထုတ်ဝေပြီး၊ developer များနှင့် အသုံးပြုသူများ စမ်းသပ်နိုင်ရန် openSUSE အမည်ရှိ အခမဲ့ distribution ကိုလည်း ၂၀၀၄ ခုနှစ်တွင် ထုတ်ဝေခဲ့သည်။

သီးခြားလွတ်လပ်သော distribution များလည်း ရှိပါသေးသည်။ ဥပမာ- အလွန်လုံခြုံစိတ်ချရသော desktop အတွက် QubesOS နှင့် software များ၏ အားနည်းချက်များကို စမ်းသပ်ရန် (penetration testing) အတွက် Kali Linux တို့ ဖြစ်ကြသည်။ ထို့ပြင် Docker ကဲ့သို့သော Linux containers များတွင် အသုံးပြုရန် အလွန်သေးငယ်သော distribution များနှင့် embedded system များအတွက် သီးသန့်ထုတ်လုပ်ထားသော distribution များလည်း ရှိသည်။

### Embedded Systems (မြှုပ်နှံထားသော စနစ်များ)

Embedded system ဆိုသည်မှာ ပိုမိုကြီးမားသော စနစ်တစ်ခုအတွင်း သတ်မှတ်ထားသော လုပ်ဆောင်ချက်တစ်ခုကို လုပ်ဆောင်ရန် ဒီဇိုင်းထုတ်ထားသည့် ကွန်ပျူတာ hardware နှင့် software ပေါင်းစပ်မှုဖြစ်သည်။ ၎င်းတို့ကို များသောအားဖြင့် အခြားစက်ပစ္စည်းများ၏ အစိတ်အပိုင်းအဖြစ် တွေ့ရလေ့ရှိပြီး ထိုစက်ပစ္စည်းများကို ထိန်းချုပ်ရန် ကူညီပေးသည်။ Embedded system များကို မော်တော်ကား၊ ဆေးဘက်ဆိုင်ရာနှင့် စစ်ဘက်ဆိုင်ရာ အသုံးချမှုများတွင် တွေ့ရှိနိုင်သည်။ Linux kernel အပေါ် အခြေခံထားသော operating system အမျိုးမျိုးကို embedded system များတွင် အသုံးပြုရန် ဖော်ဆောင်ခဲ့ကြသည်။ စမတ်ပစ္စည်းအများစုတွင် Linux kernel အခြေခံ operating system များ ပါဝင်ပတ်သက်နေသည်။

Linux ကို အခြားသော မူပိုင်ခွင့်ရှိသည့် (proprietary) embedded software များထက် ပိုမိုရွေးချယ်ရသည့် အဓိကအားသာချက်များမှာ vendor အမျိုးမျိုး၏ platform များတွင် အသုံးပြုနိုင်ခြင်း၊ ပံ့ပိုးမှု (support) ကောင်းမွန်ခြင်းနှင့် လိုင်စင်ကြေး ပေးဆောင်ရန်မလိုခြင်းတို့ ဖြစ်သည်။ လူကြိုက်အများဆုံး embedded software စီမံကိန်းနှစ်ခုမှာ ဖုန်းအမျိုးမျိုးတွင် အသုံးပြုသော Android နှင့် Raspberry Pi တွင် အသုံးပြုသော Raspbian တို့ ဖြစ်ကြသည်။

#### Android
Android သည် Google မှ အဓိကဖော်ဆောင်သော mobile operating system ဖြစ်သည်။ ၎င်းကို ၂၀၀၃ ခုနှစ်တွင် စတင်တည်ထောင်ခဲ့ပြီး မူလက digital ကင်မရာများအတွက် ရည်ရွယ်ခဲ့သော်လည်း ၂၀၀၅ ခုနှစ်တွင် Google မှ ဝယ်ယူပြီးနောက် အကြီးမားဆုံးသော mobile operating system တစ်ခုဖြစ်လာခဲ့သည်။ Android ၏ အခြေခံမှာ ပြုပြင်မွမ်းမံထားသော Linux kernel နှင့် အခြား open source software များ ပေါင်းစပ်ထားခြင်းဖြစ်သည်။ ၎င်းကို အဓိကအားဖြင့် touchscreen စက်ပစ္စည်းများအတွက် ဖန်တီးထားသော်လည်း TV၊ လက်ပတ်နာရီ၊ ဂိမ်းစက်နှင့် PC များအတွက်လည်း ဗားရှင်းအမျိုးမျိုး ရှိသည်။

Android ကို Android Open Source Project (AOSP) အဖြစ် အခမဲ့ရရှိနိုင်သော်လည်း Google သည် ၎င်း၏ ကိုယ်ပိုင် software များဖြစ်သည့် Google Calendar, Maps, Gmail, Chrome နှင့် Play Store တို့ကို မူပိုင် software များအဖြစ် ထပ်မံထည့်သွင်းပေးလေ့ရှိသည်။

#### Raspbian နှင့် Raspberry Pi
Raspberry Pi သည် ခရက်ဒစ်ကတ်အရွယ်အစားရှိ ဈေးနှုန်းချိုသာသော ကွန်ပျူတာဖြစ်ပြီး desktop ကွန်ပျူတာတစ်ခုကဲ့သို့ လုပ်ဆောင်နိုင်သလို embedded Linux system တစ်ခုအဖြစ်လည်း အသုံးပြုနိုင်သည်။ ၎င်းကို UK အခြေစိုက် Raspberry Pi Foundation မှ ပညာရေးရည်ရွယ်ချက်ဖြင့် ထုတ်လုပ်ခဲ့ခြင်းဖြစ်သည်။ Raspberry Pi တွင် အီလက်ထရွန်နစ်ပစ္စည်းများနှင့် ချိတ်ဆက်နိုင်သည့် General Purpose Input-Output (GPIO) pin များ ပါဝင်သောကြောင့် hardware ဖော်ဆောင်ရေးအတွက် ကောင်းမွန်သော platform တစ်ခု ဖြစ်လာသည်။

Raspberry Pi အတွက် အထင်ရှားဆုံး Linux distribution မှာ Raspbian ဖြစ်ပြီး ၎င်းသည် Debian မျိုးနွယ်စုဝင် ဖြစ်သည်။ ၎င်းကို Raspberry Pi hardware အတွက် အထူးပြုလုပ်ထားပြီး optimized လုပ်ထားသော package ပေါင်း ၃၅,၀၀၀ ကျော် ပါဝင်သည်။

### Linux and the Cloud (ကလောက်ပေါ်ရှိ Linux)

Cloud computing ဆိုသည်မှာ ကွန်ပျူတာ အရင်းအမြစ် (computing resources) များကို အများပြည်သူသုံး cloud (public cloud) မှ ဝယ်ယူခြင်း သို့မဟုတ် ကိုယ်ပိုင် cloud (private cloud) အဖြစ် အသုံးပြုခြင်းကို ဆိုလိုသည်။ ၂၀၁၇ ခုနှစ် အစီရင်ခံစာများအရ public cloud လုပ်ငန်းစဉ်များ၏ ၉၀% သည် Linux ပေါ်တွင် အလုပ်လုပ်နေခြင်းဖြစ်သည်။

Amazon Web Services (AWS), Google Cloud Platform (GCP) နှင့် Microsoft Azure ကဲ့သို့သော cloud provider အားလုံးသည် Linux ဗားရှင်းအမျိုးမျိုးကို ပံ့ပိုးပေးထားသည်။ Linux ကို များသောအားဖြင့် Infrastructure as a Service (IaaS) ပုံစံဖြင့် ပံ့ပိုးပေးလေ့ရှိပြီး၊ အသုံးပြုသူများသည် မိနစ်ပိုင်းအတွင်း virtual machine များကို စတင်အသုံးပြုနိုင်သည်။ Cloud အသုံးပြုသူသည် မိမိနှစ်သက်ရာ distribution ပါဝင်သော image ကို ရွေးချယ်ပြီး လျင်မြန်စွာ cloud instance တစ်ခုကို တည်ဆောက်နိုင်သည်။ Cloud provider များသည် ၎င်းတို့၏ image များတွင် cloud ပတ်ဝန်းကျင်နှင့် ကိုက်ညီအောင် ညှိနှိုင်းပေးနိုင်သော tool များကိုလည်း ထည့်သွင်းပေးထားလေ့ရှိသည်။
